# Tidy Data — Hadley Wickham

## 결론부터

이 논문의 결론은 아주 단순합니다.

**데이터 분석을 잘하려면, 모델을 돌리기 전에 데이터를 “분석하기 좋은 표준 형태”로 정리해야 한다.**  
그 표준 형태가 바로 **tidy data**입니다.

Hadley Wickham이 말하는 tidy data는 다음 3가지 조건을 만족하는 데이터입니다.

| 원칙 | 쉽게 말하면 |
|---|---|
| **각 변수는 하나의 열 column** | 키, 몸무게, 날짜, 매출액 같은 정보는 각각 별도 컬럼이어야 함 |
| **각 관측치는 하나의 행 row** | 한 사람, 한 거래, 하루, 한 실험 결과는 한 줄이어야 함 |
| **각 관측 단위는 하나의 테이블** | 고객 정보, 주문 정보, 상품 정보처럼 성격이 다른 것은 테이블을 나눠야 함 |

이 논문은 2014년 *Journal of Statistical Software*에 실린 논문이고, 핵심 주장은 “정돈된 데이터 구조를 쓰면 데이터를 조작하고, 모델링하고, 시각화하기 쉬워진다”는 것입니다. 논문 초록에서도 tidy data의 구조를 “각 변수는 열, 각 관측치는 행, 각 관측 단위는 테이블”이라고 명확히 정의합니다.

---

## 이 논문을 한 문장으로 요약하면

**데이터 분석의 생산성은 좋은 모델보다도, 먼저 좋은 데이터 구조에서 나온다.**

이 논문이 중요한 이유는 “데이터 클리닝”이라는 넓은 문제 중에서도, 특히 **데이터의 모양을 어떻게 잡을 것인가**를 표준화했기 때문입니다. Wickham은 데이터 준비가 분석 과정에서 반복적으로 일어나며, 이 반복 작업을 줄이려면 데이터를 일관된 구조로 만들어야 한다고 설명합니다.

---

## 왜 이게 데이터사이언스 입문에서 중요한가?

초보자는 보통 이런 순서로 생각합니다.

> “좋은 모델을 쓰면 좋은 분석이 되겠지?”

그런데 실제 데이터 분석에서는 이런 일이 훨씬 자주 생깁니다.

> “모델은 아직 돌리지도 못했는데, 데이터 모양이 이상해서 전처리만 계속하고 있다.”

예를 들어 이런 표가 있다고 해보겠습니다.

| 이름 | 2023년 매출 | 2024년 매출 |
|---|---:|---:|
| A고객 | 100 | 130 |
| B고객 | 80 | 90 |

겉보기에는 보기 좋습니다. 하지만 분석하기에는 애매합니다. 왜냐하면 `2023년 매출`, `2024년 매출`이라는 컬럼명 안에 사실상 **연도**라는 값이 숨어 있기 때문입니다.

tidy data 관점에서는 이렇게 바꾸는 것이 좋습니다.

| 이름 | 연도 | 매출 |
|---|---:|---:|
| A고객 | 2023 | 100 |
| A고객 | 2024 | 130 |
| B고객 | 2023 | 80 |
| B고객 | 2024 | 90 |

이렇게 바꾸면 “연도별 매출 평균”, “고객별 성장률”, “2024년만 필터링”, “시각화” 같은 작업이 훨씬 쉬워집니다.

즉, **사람이 보기 좋은 표**와 **컴퓨터가 분석하기 좋은 표**는 다를 수 있습니다. 이 논문은 후자를 어떻게 만들지 설명합니다.

---

## 핵심 개념 1: 변수, 관측치, 값

이 논문을 이해하려면 세 단어만 잡으면 됩니다.

### 1. 변수 variable

변수는 우리가 측정하거나 기록한 속성입니다.

예를 들면:

| 변수 예시 |
|---|
| 이름 |
| 나이 |
| 성별 |
| 날짜 |
| 매출 |
| 온도 |
| 치료법 |
| 실험 결과 |

데이터프레임에서는 보통 **컬럼**이 되어야 합니다.

### 2. 관측치 observation

관측치는 “하나의 분석 단위”입니다.

예를 들면:

| 상황 | 관측치 |
|---|---|
| 고객 분석 | 고객 한 명 |
| 매출 분석 | 거래 한 건 |
| 시계열 분석 | 하루 또는 한 시점 |
| 실험 분석 | 한 사람에게 한 처치를 적용한 결과 |
| 웹 로그 분석 | 클릭 한 번 또는 세션 하나 |

tidy data에서는 관측치 하나가 **행 하나**가 됩니다.

### 3. 값 value

값은 실제 데이터 셀에 들어가는 내용입니다.

예를 들면:

| 변수 | 값 |
|---|---|
| 나이 | 27 |
| 성별 | 여성 |
| 날짜 | 2026-05-19 |
| 매출 | 150000 |
| 치료법 | A |

논문에서는 데이터셋을 값들의 모음으로 보고, 각 값은 어떤 **변수**에 속하면서 동시에 어떤 **관측치**에도 속한다고 설명합니다.

---

## 핵심 개념 2: “지저분한 데이터”의 대표 패턴

Wickham은 현실의 데이터가 보통 tidy하지 않다고 말하면서, 흔한 messy data 패턴을 정리합니다. 대표적으로 다음과 같은 경우입니다.

| 지저분한 패턴 | 쉬운 설명 |
|---|---|
| 컬럼명이 변수명이 아니라 값이다 | `2023`, `2024`가 컬럼명으로 들어가 있음 |
| 한 컬럼에 여러 변수가 섞여 있다 | `남_20대`, `여_30대`처럼 성별과 연령대가 한 컬럼에 섞임 |
| 변수가 행과 열 양쪽에 흩어져 있다 | 어떤 변수는 행에, 어떤 변수는 열에 있음 |
| 여러 관측 단위가 한 테이블에 섞여 있다 | 고객 정보와 주문 정보가 한 표에 뒤섞임 |
| 하나의 관측 단위가 여러 테이블에 찢어져 있다 | 같은 고객 정보가 여러 파일에 나뉘어 있음 |

이 논문의 강점은 “데이터가 왜 지저분한가?”를 감각적으로 설명하는 데 그치지 않고, **어떤 구조가 분석하기 좋은 구조인지 기준을 준다**는 데 있습니다.

---

## 쉬운 예시: wide format vs long format

데이터 분석을 하다 보면 자주 만나는 차이가 있습니다.

### Wide format

| 학생 | 수학 | 영어 | 과학 |
|---|---:|---:|---:|
| 민수 | 90 | 80 | 70 |
| 지영 | 85 | 95 | 90 |

이 표는 사람이 보기에는 좋습니다. 하지만 “과목별 평균”, “점수 분포”, “과목을 x축으로 한 시각화”를 하려면 불편할 수 있습니다.

### Tidy / long format

| 학생 | 과목 | 점수 |
|---|---|---:|
| 민수 | 수학 | 90 |
| 민수 | 영어 | 80 |
| 민수 | 과학 | 70 |
| 지영 | 수학 | 85 |
| 지영 | 영어 | 95 |
| 지영 | 과학 | 90 |

이 구조에서는 `학생`, `과목`, `점수`가 각각 변수입니다. 관측치 하나는 “어떤 학생의 어떤 과목 점수”입니다.

즉, tidy data는 무조건 짧고 예쁜 표가 아니라, **분석 단위가 명확한 표**입니다.

## 이 논문이 말하는 진짜 메시지

표면적으로는 “데이터를 이렇게 정리하라”는 논문처럼 보입니다. 하지만 더 깊게 보면 이 논문은 **데이터 분석 워크플로우의 철학**을 말합니다.

논문의 진짜 메시지는 이것입니다.

> 데이터를 매번 임기응변으로 고치지 말고,  
> 분석 도구들이 잘 작동할 수 있는 공통 형식으로 맞춰라.

이게 중요한 이유는, 데이터가 tidy하면 다음 작업들이 자연스럽게 이어지기 때문입니다.

| 작업 | tidy data가 주는 장점 |
|---|---|
| 필터링 | 특정 조건의 행만 쉽게 고를 수 있음 |
| 집계 | 그룹별 평균, 합계, 개수를 쉽게 계산할 수 있음 |
| 시각화 | x축, y축, 색상, 그룹을 변수로 바로 매핑할 수 있음 |
| 모델링 | 입력 변수와 타깃 변수를 명확히 분리할 수 있음 |
| 재사용 | 한 번 정리한 데이터를 여러 도구에서 반복 사용 가능 |

Wickham은 tidy data와 tidy tools가 함께 작동할 때 분석자가 데이터 조작의 잡무보다 실제 도메인 문제에 집중할 수 있다고 설명합니다.

---

## 이 논문을 공부할 때 가장 중요한 질문

이 논문을 읽으면서 계속 던져야 할 질문은 하나입니다.

**“이 데이터에서 관측치 하나는 정확히 무엇인가?”**

이 질문이 제일 중요합니다.

예를 들어 매출 데이터를 본다고 해도 관측치가 무엇인지에 따라 데이터 구조가 달라집니다.

| 분석 목적 | 관측치 |
|---|---|
| 고객별 총매출 분석 | 고객 한 명 |
| 일별 매출 추이 분석 | 날짜 하루 |
| 상품별 판매량 분석 | 상품 하나 |
| 주문 로그 분석 | 주문 한 건 |
| 주문 상세 분석 | 주문-상품 조합 한 건 |

관측치를 잘못 잡으면 컬럼도 잘못 만들고, 집계도 잘못하고, 모델도 이상해집니다.

---

## 입문자가 가져가야 할 실전 규칙

이 논문을 읽고 바로 적용할 규칙은 다음 5개입니다.

| 규칙 | 설명 |
|---|---|
| 1 | 컬럼명에는 값이 아니라 변수명을 넣는다 |
| 2 | 한 컬럼에는 한 종류의 정보만 넣는다 |
| 3 | 한 행은 하나의 관측치를 나타내게 한다 |
| 4 | 서로 다른 관측 단위는 테이블을 나눈다 |
| 5 | 분석 전에 “관측치가 무엇인가?”를 먼저 정의한다 |

예를 들어 `남성_20대_서울`이라는 컬럼이나 값이 있다면, 대개 tidy하지 않은 신호입니다. 보통은 `성별`, `연령대`, `지역`이라는 세 변수로 분리하는 편이 좋습니다.

---

## 이 논문의 한계도 알아야 함

중요한 점은, tidy data가 **항상 최고의 저장 방식**은 아니라는 것입니다.

예를 들어 대규모 행렬 연산, 이미지 데이터, 시계열 배열, 딥러닝 텐서, 희소 행렬 같은 경우에는 tidy dataframe보다 matrix나 array 형태가 더 효율적일 수 있습니다. Wickham도 논문 후반에서 tidy data가 최종 해답이라고 단정하지 않고, 다른 데이터 표현 방식과 도구의 가능성을 언급합니다.

그래서 이 논문을 이렇게 이해하면 좋습니다.

**tidy data는 모든 데이터를 저장하는 유일한 정답이 아니라, 탐색·시각화·통계 분석·전처리를 쉽게 만드는 강력한 기본형이다.**
