# Training Language Models to Follow Instructions with Human Feedback — InstructGPT / RLHF

## 결론부터

이 논문의 결론은 이겁니다.

**큰 언어 모델이 다음 단어를 잘 예측한다고 해서, 사용자의 지시를 잘 따르고, 진실하고, 안전하고, 도움이 되는 답변을 한다는 보장은 없다.**

그래서 모델을 사람의 선호와 의도에 맞추기 위해 **사람의 피드백을 이용해 추가 학습**해야 한다.

이 방식이 바로 **RLHF, Reinforcement Learning from Human Feedback**입니다.

한 줄로 말하면:

> **InstructGPT는 GPT-3를 사람이 원하는 답변 방식에 맞게 조정한 모델이고, 그 핵심 방법이 RLHF입니다.**

논문은 큰 언어 모델이 사용자의 의도를 더 잘 따른다고 자동으로 보장되지 않으며, 거짓이거나 유해하거나 도움이 되지 않는 출력을 만들 수 있다고 지적합니다. 저자들은 사람의 피드백을 이용해 GPT-3를 fine-tuning함으로써 사용자 의도에 더 잘 맞는 모델을 만들 수 있음을 보였고, 그 결과 모델을 **InstructGPT**라고 불렀습니다.

---

## 이 논문을 한 문장으로 요약하면

**InstructGPT는 “다음 토큰을 잘 맞히는 모델”을 “사용자 지시를 잘 따르는 모델”로 바꾸기 위해, 사람이 좋은 답변을 보여주고, 여러 답변 중 더 나은 답을 고르고, 그 선호를 보상으로 삼아 모델을 학습시킨 논문입니다.**

이전 논문들과 연결하면 흐름이 이렇습니다.

| 논문 | 핵심 |
|---|---|
| **GPT-3** | 거대한 언어 모델은 prompt만 보고 다양한 task를 수행할 수 있다 |
| **Scaling Laws / Chinchilla** | 모델 크기, token 수, compute를 잘 배분하면 성능이 좋아진다 |
| **RAG** | 모델 내부 지식만 믿지 말고 외부 문서를 검색해 답하자 |
| **InstructGPT / RLHF** | 모델이 사용자의 의도를 더 잘 따르도록 사람의 피드백으로 조정하자 |

GPT-3는 다음 token 예측을 매우 잘했습니다. 하지만 다음 token 예측은 본질적으로 이런 목표입니다.

```text
인터넷 텍스트에서 다음에 올 법한 말을 예측하라.
```

사용자가 원하는 목표는 다릅니다.

```text
내 질문에 정확하고, 도움이 되고, 안전하게 답하라.
```

이 둘은 같지 않습니다.

---

## 왜 InstructGPT가 필요했나?

GPT-3 같은 모델은 prompt를 잘 쓰면 많은 일을 할 수 있습니다.

하지만 모델이 항상 사용자의 의도를 제대로 따르지는 않습니다.

예를 들어 사용자가 이렇게 물었다고 합시다.

```text
p-value를 고등학생도 이해할 수 있게 설명해줘.
```

기존 GPT-3는 내용은 맞을 수 있지만 너무 어렵거나, 사용자의 “고등학생도 이해할 수 있게”라는 지시를 잘 따르지 못할 수 있습니다.

또는 이렇게 물었다고 합시다.

```text
이 문장을 세 줄로 요약해줘.
```

모델이 다섯 줄로 답하거나, 요약이 아니라 해설을 길게 붙일 수 있습니다.

즉, 문제는 단순히 “지식을 아느냐”가 아닙니다.

> **사용자가 원하는 형식, 난이도, 태도, 안전성, 진실성을 맞추느냐**가 문제입니다.

---

## 핵심 개념 1: Instruction Following

### 결론부터

**Instruction following은 사용자가 자연어로 지시한 일을 모델이 그대로 이해하고 수행하는 능력입니다.**

예를 들면 이런 것들입니다.

```text
- 이 글을 세 문장으로 요약해줘.
- 초등학생도 이해하게 설명해줘.
- 표로 정리해줘.
- 코드에 주석을 달아줘.
- 반말을 존댓말로 바꿔줘.
- 논문 결론을 먼저 설명해줘.
```

GPT-3도 이런 일을 어느 정도 할 수 있었지만, InstructGPT는 이런 사용자 지시를 더 안정적으로 따르도록 학습되었습니다.

여기서 중요한 점은, instruction following은 단순한 정답 맞히기가 아닙니다.

같은 질문에도 지시가 다르면 답이 달라져야 합니다.

```text
"p-value를 한 문장으로 설명해줘"
"p-value를 예시와 함께 길게 설명해줘"
"p-value를 수식 중심으로 설명해줘"
"p-value를 비전공자에게 설명해줘"
```

좋은 instruction-following 모델은 질문의 내용뿐 아니라 **사용자가 원하는 답변 방식**도 이해해야 합니다.

---

## 핵심 개념 2: Alignment

### 결론부터

**Alignment는 모델의 행동이 사람의 의도와 선호에 맞도록 만드는 것입니다.**

여기서 말하는 alignment는 거창한 철학 문제만이 아닙니다. 이 논문에서는 실용적으로 이런 것들을 의미합니다.

| 목표 | 의미 |
|---|---|
| **Helpful** | 사용자의 과제를 실제로 도와야 함 |
| **Honest** | 모르는 것은 모른다고 하고, 사실을 지어내지 않아야 함 |
| **Harmless** | 유해하거나 위험한 출력을 줄여야 함 |

예를 들어 사용자가 묻습니다.

```text
이 약을 하루에 몇 알 먹어야 하나요?
```

모델이 그럴듯한 복용량을 지어내면 위험합니다.

좋은 모델은 상황에 따라 이렇게 답해야 합니다.

```text
정확한 복용량은 약 이름, 용량, 처방 여부, 건강 상태에 따라 다르므로 의사나 약사에게 확인해야 합니다.
```

즉, alignment는 단순히 답을 “많이” 하는 것이 아니라, **사용자에게 도움이 되는 방향으로 적절하게 답하는 것**입니다.

---

## 핵심 개념 3: RLHF의 3단계

InstructGPT의 핵심은 세 단계입니다.

```text
1. SFT: 사람이 좋은 답변을 직접 써서 모델을 지도학습
2. RM: 사람이 여러 답변을 비교해 순위를 매기고, reward model 학습
3. PPO: reward model 점수가 높아지도록 언어 모델을 강화학습
```

---

## 1단계: SFT — Supervised Fine-Tuning

### 결론부터

**SFT는 사람이 직접 “좋은 답변 예시”를 써주고, 모델이 그 답변을 따라 하도록 학습시키는 단계입니다.**

예를 들어 prompt가 있습니다.

```text
사용자:
p-value를 초보자도 이해할 수 있게 설명해줘.
```

사람 labeler가 좋은 답변을 작성합니다.

```text
p-value는 “귀무가설이 맞다고 가정했을 때, 지금 관측한 결과만큼 극단적인 결과가 나올 확률”입니다...
```

이렇게 prompt와 좋은 답변 쌍을 만듭니다.

```text
입력: p-value를 초보자도 이해할 수 있게 설명해줘.
정답: 사람이 작성한 좋은 설명
```

그다음 GPT-3를 이 데이터로 supervised fine-tuning합니다.

즉, 모델에게 이렇게 가르칩니다.

> “이런 지시가 들어오면, 이런 식으로 답하는 게 좋다.”

---

## SFT를 쉬운 비유로 보면

GPT-3는 글을 잘 쓰는 학생입니다.

하지만 선생님이 원하는 답안 형식을 모릅니다.

그래서 선생님이 모범답안을 보여줍니다.

```text
문제: 이 문단을 요약하라.
모범답안: ...
```

학생은 모범답안을 따라 하며 배웁니다.

이게 SFT입니다.

---

## 2단계: Reward Model 학습

### 결론부터

**Reward Model은 “어떤 답변이 사람이 더 좋아할 답변인지 점수로 예측하는 모델”입니다.**

SFT만으로도 모델은 좋아집니다.

하지만 사람이 모든 prompt에 대해 완벽한 정답을 직접 쓰는 것은 너무 비쌉니다.

그래서 다음 단계에서는 사람에게 답변을 직접 쓰게 하기보다, 여러 답변을 비교하게 합니다.

예를 들어 같은 질문에 대해 모델이 답변 4개를 만들었다고 합시다.

```text
질문:
p-value를 쉽게 설명해줘.

답변 A
답변 B
답변 C
답변 D
```

사람 labeler는 이 답변들을 순위로 매깁니다.

```text
1등: B
2등: D
3등: A
4등: C
```

이제 이 ranking 데이터를 이용해 reward model을 학습합니다.

Reward model은 이런 일을 배웁니다.

```text
질문 + 답변 → 사람이 좋아할 점수
```

예를 들어:

| 답변 | Reward Model 점수 |
|---|---:|
| B | 8.7 |
| D | 7.2 |
| A | 4.1 |
| C | 1.3 |

Reward model은 사람 피드백을 대체하는 **학습용 채점기**입니다.

---

## 3단계: PPO — Reinforcement Learning

### 결론부터

**PPO 단계에서는 언어 모델이 reward model에게 높은 점수를 받는 답변을 만들도록 추가 학습합니다.**

이제 모델은 답변을 생성합니다.

```text
질문: p-value를 쉽게 설명해줘.
모델 답변: ...
```

Reward model이 점수를 줍니다.

```text
점수: 7.8
```

모델은 더 높은 점수를 받는 방향으로 업데이트됩니다.

```text
답변이 사람이 선호하는 방향에 가까우면 보상 증가
그렇지 않으면 보상 감소
```

이때 사용한 강화학습 알고리즘이 **PPO, Proximal Policy Optimization**입니다.

PPO는 모델이 한 번에 너무 크게 변하지 않도록 비교적 안정적으로 업데이트하는 강화학습 방법입니다.

---

## 전체 흐름을 하나의 예시로 보기

## 원래 GPT-3

사용자:

```text
이 글을 세 문장으로 요약해줘.
```

GPT-3는 답할 수는 있지만, 지시를 정확히 안 따를 수 있습니다.

```text
요약이 다섯 문장이다.
중요하지 않은 이야기를 덧붙인다.
말투가 들쭉날쭉하다.
```

## 1단계 SFT

사람이 좋은 답변 예시를 써줍니다.

```text
입력: 이 글을 세 문장으로 요약해줘.
좋은 답변: 정확히 세 문장으로 핵심만 요약
```

모델은 이 답변을 따라 하도록 학습합니다.

## 2단계 Reward Model

같은 prompt에 대해 여러 답변을 만듭니다.

```text
답변 A: 세 문장, 정확함
답변 B: 한 문장, 너무 짧음
답변 C: 다섯 문장, 지시 위반
답변 D: 세 문장이지만 부정확함
```

사람이 순위를 매깁니다.

```text
A > D > B > C
```

Reward model은 이 선호를 학습합니다.

## 3단계 PPO

모델이 답변을 생성합니다.

Reward model이 점수를 줍니다.

모델은 점수가 높은 답변을 더 자주 생성하도록 업데이트됩니다.

---

## 핵심 개념 4: RLHF는 “정답”보다 “선호”를 학습한다

많은 언어 문제에는 하나의 정답만 있는 것이 아닙니다.

예를 들어 사용자가 묻습니다.

```text
좋은 자기소개서를 쓰는 법을 알려줘.
```

가능한 좋은 답변은 여러 개입니다.

```text
답변 A: 구조 중심으로 설명
답변 B: 예시 중심으로 설명
답변 C: 체크리스트로 설명
```

모두 괜찮을 수 있습니다.

그러면 “정답 하나”를 맞히는 방식은 한계가 있습니다.

RLHF는 이렇게 접근합니다.

```text
여러 답변 중 사람이 더 선호하는 답변을 학습하자.
```

즉, RLHF는 정답 label보다 **human preference**를 중심에 둡니다.

---

## SFT와 RLHF의 차이

| 구분 | SFT | RLHF |
|---|---|---|
| 학습 데이터 | 사람이 쓴 좋은 답변 | 사람이 여러 답변에 매긴 선호 순위 |
| 모델이 배우는 것 | 모범답안 따라 하기 | 사람이 선호할 답변 만들기 |
| 방식 | supervised learning | reward model + reinforcement learning |
| 장점 | 단순하고 안정적 | 선호와 품질을 더 직접적으로 반영 |
| 단점 | 좋은 답변을 직접 써야 함 | reward model 오류, RL 불안정 가능 |

SFT는 “이렇게 답해라”를 배우는 단계입니다.

RLHF는 “여러 답변 중 사람이 더 좋아할 답변을 만들어라”를 배우는 단계입니다.

---

## 왜 그냥 SFT만 하면 안 되나?

SFT만 해도 모델은 좋아집니다.

하지만 SFT에는 한계가 있습니다.

## 1. 모범답안을 직접 쓰는 비용이 큼

사람이 모든 prompt에 대해 좋은 답변을 쓰는 것은 비쌉니다.

## 2. 좋은 답변이 하나로 정해져 있지 않음

같은 질문에도 좋은 답변은 여러 가지입니다.

## 3. 모델이 생성한 답변의 미묘한 차이를 반영하기 어려움

예를 들어 두 답변이 있습니다.

```text
A: 정확하지만 너무 딱딱함
B: 정확하고 친절함
```

SFT는 이런 미묘한 선호를 직접 반영하기 어렵습니다.

Reward model은 이런 선호 비교 데이터를 학습합니다.

---

## 핵심 개념 5: Alignment Tax

### 결론부터

**Alignment tax는 모델을 더 안전하고 지시를 잘 따르게 만들면서, 기존 benchmark 성능이 떨어지는 비용을 말합니다.**

모델을 alignment 방향으로 fine-tuning하면 이런 걱정이 생깁니다.

```text
사용자 지시는 더 잘 따르는데,
원래 갖고 있던 NLP 능력은 떨어지는 것 아닌가?
```

이 성능 저하를 **alignment tax**라고 볼 수 있습니다.

이를 줄이기 위해 PPO 학습 중 pretraining data 일부를 섞어 일반 next-token prediction loss도 함께 학습할 수 있습니다. 이런 방식은 PPO-ptx라고 부릅니다.

## 결과: 작은 InstructGPT가 큰 GPT-3보다 선호됨

이 논문의 가장 유명한 결과는 이것입니다.

**1.3B InstructGPT가 175B GPT-3보다 labeler에게 더 선호되었습니다.**

이건 매우 큰 의미가 있습니다.

```text
모델 크기 100배 이상 차이
그런데 instruction-following 품질은 작은 InstructGPT가 더 좋음
```

이 결과가 주는 메시지는 강합니다.

> 모델을 더 크게 만드는 것만큼이나, **어떻게 사람의 의도에 맞게 조정하느냐**가 중요하다.

---

## RLHF는 새로운 지식을 많이 가르치는 방법인가?

### 결론부터

**아닙니다. RLHF는 주로 모델의 행동 방식을 조정하는 방법입니다.**

RLHF는 주로 이런 것을 바꿉니다.

```text
답변 태도
지시 준수
출력 형식
불필요한 장황함
위험한 답변 회피
사실을 지어내는 경향 감소
```

하지만 모델이 pretraining 때 전혀 배우지 못한 지식을 RLHF만으로 대량 주입하는 것은 어렵습니다.

새로운 지식이 필요하면 보통 다음 방법이 더 직접적입니다.

```text
continued pretraining
fine-tuning
RAG
tool use
database 연결
```

---

## InstructGPT와 RAG의 차이

| 구분 | RAG | InstructGPT / RLHF |
|---|---|---|
| 핵심 질문 | 필요한 지식을 어디서 가져올까? | 답변을 어떻게 사람 의도에 맞출까? |
| 방법 | 외부 문서 검색 + 생성 | 사람 피드백으로 모델 행동 조정 |
| 바꾸는 것 | 입력 context에 지식 추가 | 모델의 응답 습관과 선호 |
| 강점 | 최신성, 근거, 도메인 문서 활용 | 지시 준수, helpfulness, 답변 품질 |
| 한계 | 검색 실패 시 문제 | reward model 오류, 선호 편향 |

실무에서는 둘을 같이 씁니다.

```text
RAG로 관련 문서를 찾아오고,
RLHF된 모델이 그 문서를 바탕으로 사용자에게 잘 답한다.
```

---

## GPT-3, RAG, InstructGPT를 한 번에 비교

| 모델/방법 | 무엇을 잘하게 하나? | 핵심 수단 |
|---|---|---|
| **GPT-3** | 다음 token 생성, few-shot prompting | 대규모 pretraining |
| **RAG** | 외부 지식 기반 답변 | 검색 + 생성 |
| **InstructGPT** | 사용자 지시 따르기 | SFT + RM + PPO/RLHF |

예를 들어 사용자가 묻습니다.

```text
우리 회사의 육아휴직 규정을 요약해줘.
```

GPT-3만 있으면 내부 규정을 모를 수 있습니다.

RAG를 붙이면 회사 문서를 찾아올 수 있습니다.

InstructGPT식 alignment가 되어 있으면 그 문서를 사용자의 요구에 맞게 친절하고 정확한 형식으로 요약할 가능성이 높아집니다.

## 이 논문의 한계

## 1. InstructGPT도 여전히 틀린다

InstructGPT가 지시를 따르지 못하거나, 사실을 지어내거나, false premise가 있는 instruction을 제대로 감지하지 못하거나, 간단한 질문에 장황하게 회피하는 답을 할 수 있습니다.

```text
hallucination 감소 가능
하지만 제거는 아님
```

## 2. 사람 피드백은 특정 집단의 선호를 반영한다

사람 피드백은 객관적인 절대 진리가 아닙니다.

어떤 labeler가 평가했는지, 어떤 지침을 받았는지, 어떤 문화적 배경을 갖고 있는지에 영향을 받습니다.

## 3. Reward model을 속이는 문제가 생길 수 있다

Reward model은 사람 선호를 완벽히 대변하지 않습니다.

모델이 reward model 점수를 높이는 방향으로 학습하다 보면, 실제 사람에게 좋은 답변이 아니라 reward model이 좋아하는 패턴을 과도하게 최적화할 수 있습니다.

## 4. RLHF는 비용이 든다

RLHF에는 사람이 필요합니다.

```text
좋은 답변 작성
답변 비교 평가
labeler 교육
품질 관리
reward model 학습
PPO 학습
```

그래서 데이터 수집과 운영 비용이 큽니다.

---

## 이 논문이 현대 LLM에 준 영향

InstructGPT는 현대 instruction-following LLM의 핵심 흐름을 만들었습니다.

이후 많은 모델이 비슷한 단계를 따릅니다.

```text
1. Pretraining
2. Supervised instruction tuning
3. Preference optimization / RLHF / DPO류 방법
```

InstructGPT 논문은 특히 다음 관점을 강하게 만들었습니다.

```text
대규모 pretraining만으로는 부족하다.
사용자 의도에 맞추는 post-training이 중요하다.
```

요즘 LLM 개발에서 흔히 말하는 **post-training**의 핵심이 바로 이 흐름입니다.

---

## 입문자가 꼭 기억해야 할 문장

**InstructGPT는 GPT-3를 사람의 demonstration과 preference feedback으로 fine-tuning해서, 사용자의 지시를 더 잘 따르고 더 도움이 되는 답변을 하도록 만든 모델입니다.**

더 짧게 말하면:

> **InstructGPT / RLHF = 사람이 좋아하는 답변 방식을 모델에게 가르치는 방법**

## 오늘 공부용 요약

**논문명:** Training Language Models to Follow Instructions with Human Feedback  
**저자:** Long Ouyang et al.  
**핵심 결론:** GPT-3 같은 큰 언어 모델은 다음 token 예측을 잘해도 사용자 의도에 맞게 답한다고 보장할 수 없다. 사람의 demonstration과 preference ranking을 이용해 SFT, reward model, PPO/RLHF 단계를 거치면 instruction-following, truthfulness, toxicity 측면에서 개선된 InstructGPT를 만들 수 있다.

**왜 중요함:** 이 논문은 “pretraining으로 큰 모델을 만든 뒤, 사람 피드백으로 post-training해서 assistant처럼 동작하게 만든다”는 현대 LLM 개발 흐름의 핵심 논문입니다.

**입문자가 배울 점:** RLHF는 모델에게 새로운 지식을 많이 넣는 방법이라기보다, 모델이 이미 가진 언어 능력을 사용자 의도와 사람 선호에 맞게 끌어내는 alignment 방법입니다.

**가장 중요한 문장:** InstructGPT는 “더 큰 모델”보다 “사람이 원하는 방식으로 답하도록 조정된 모델”이 실제 사용자에게 더 선호될 수 있음을 보여줬다.
