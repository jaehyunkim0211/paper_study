# Self-Consistency Improves Chain of Thought Reasoning in Language Models

## 결론부터

이 논문의 결론은 이겁니다.

**LLM에게 Chain-of-Thought로 한 번만 풀이하게 하지 말고, 여러 개의 풀이 과정을 샘플링한 뒤 가장 많이 나온 최종 답을 고르면 reasoning 성능이 더 좋아진다.**

한 줄로 말하면:

> **Self-Consistency = 여러 풀이를 만들게 하고, 가장 일관되게 나온 답을 선택하는 방법**

예를 들어 같은 문제를 여러 번 풀게 했을 때:

```text
풀이 1 → 답: 11
풀이 2 → 답: 11
풀이 3 → 답: 9
풀이 4 → 답: 11
풀이 5 → 답: 13
```

가장 많이 나온 답은 11입니다.

```text
최종 답: 11
```

---

## 이 논문을 한 문장으로 요약하면

**복잡한 문제에는 여러 올바른 풀이 경로가 있을 수 있으므로, 하나의 풀이만 믿지 말고 여러 풀이가 합의하는 답을 고르자는 논문입니다.**

CoT는 “단계별로 풀게 하자”였습니다. Self-Consistency는 한 단계 더 나아갑니다.

```text
중간 풀이를 하나만 만들지 말고,
여러 개 만들게 하자.
그리고 가장 많이 도달한 답을 고르자.
```

---

## 왜 필요한가?

CoT는 강력하지만, 기본적으로 하나의 decoding path에 의존할 수 있습니다.

```text
가장 그럴듯한 다음 token
→ 그다음 가장 그럴듯한 token
→ 하나의 풀이
→ 하나의 답
```

문제는 첫 풀이가 틀릴 수 있다는 것입니다. 사람도 중요한 문제를 한 번만 풀지 않고, 다른 방식으로 다시 풀거나 검산합니다. Self-Consistency는 이 인간적인 검산 방식을 LLM에 적용합니다.

---

## 핵심 개념 1: Greedy Decoding

Greedy decoding은 매 순간 가장 확률이 높은 다음 token을 고르는 방식입니다.

```text
"처음에" 0.45
"먼저"   0.30
"우선"   0.15
```

항상 `처음에`를 고릅니다.

장점은 안정적이고 빠르다는 것입니다. 단점은 다른 가능한 풀이를 탐색하지 않는다는 것입니다.

---

## 핵심 개념 2: Sampling

Sampling은 항상 가장 높은 확률의 token만 고르지 않고, 확률 분포에 따라 다양한 출력을 만들게 합니다.

```text
출력 1: 처음에 ...
출력 2: 먼저 ...
출력 3: 우선 ...
```

이렇게 하면 다양한 reasoning path를 만들 수 있습니다.

---

## 핵심 개념 3: Reasoning Path

Reasoning path는 최종 답에 도달하기까지 모델이 생성한 중간 풀이 과정입니다.

예를 들어 같은 문제를 두 방식으로 풀 수 있습니다.

```text
방법 1: 두 번째 정지 지점 = 60 - 15 = 45, 45 - 20 = 25
방법 2: 전체 60에서 첫 구간 20과 마지막 구간 15를 빼면 25
```

풀이 경로는 다르지만 답은 같습니다.

---

## 핵심 개념 4: Majority Vote

Self-Consistency의 가장 단순한 집계 방법은 최종 답 다수결입니다.

```text
답 25: 3번
답 40: 1번
답 35: 1번
```

최종 답은 25입니다.

---

## 알고리즘

```text
1. CoT 예시가 들어간 prompt를 만든다.
2. 같은 문제에 대해 모델 출력을 여러 번 샘플링한다.
3. 각 출력에서 최종 답만 추출한다.
4. 가장 많이 나온 답을 최종 답으로 선택한다.
```

---

## 핵심 철학: 정답은 하나지만 풀이 경로는 여러 개

복잡한 reasoning 문제는 여러 방식으로 풀 수 있습니다.

```text
식으로 풀기
그림으로 풀기
거꾸로 검산하기
```

올바른 풀이들은 같은 정답으로 모이는 경향이 있고, 틀린 풀이들은 서로 다른 오답으로 흩어질 가능성이 높습니다.

---

## Self-Consistency는 앙상블과 비슷하다

일반 앙상블은 여러 모델을 씁니다.

```text
모델 A
모델 B
모델 C
```

Self-Consistency는 하나의 모델에서 여러 출력을 샘플링합니다.

```text
같은 모델 → 출력 1, 출력 2, 출력 3
```

즉, self-ensemble에 가깝습니다.

---

## Sample-and-Rank와 다르다

Sample-and-rank는 여러 출력을 만든 뒤 log probability가 높은 출력 하나를 고릅니다. Self-Consistency는 출력의 확률이 아니라 **최종 답의 일관성**을 봅니다.

```text
출력 1 → 답 25
출력 2 → 답 25
출력 3 → 답 40
```

답 25가 가장 자주 나오면 선택합니다.

---

## 언제 유용한가?

특히 좋은 경우:

```text
수학 word problem
논리 추론
상식 추론
객관식 reasoning
symbolic manipulation
답이 명확한 문제
```

덜 적합한 경우:

```text
창의적 글쓰기
긴 essay
요약
번역
정답이 하나로 고정되지 않은 open-ended generation
```

---

## 한계

1. **비용이 많이 듭니다.** 여러 번 생성하므로 latency와 token 비용이 증가합니다.
2. **최종 답 추출이 가능해야 합니다.** 객관식·수학·yes/no는 쉽지만 긴 서술형은 어렵습니다.
3. **틀린 답이 일관되게 나올 수도 있습니다.** 같은 오해를 반복하면 다수결도 틀립니다.
4. **작은 모델에서는 효과가 제한될 수 있습니다.** 다양한 유효 reasoning path를 만들 능력이 필요합니다.

---

## CoT와 Self-Consistency 차이

| 구분 | Chain-of-Thought | Self-Consistency |
|---|---|---|
| 핵심 | 중간 풀이를 쓰게 함 | 여러 중간 풀이를 샘플링하고 답을 집계 |
| 출력 수 | 보통 1개 | 여러 개 |
| 최종 답 선택 | 한 풀이의 답 | 여러 풀이의 다수결 |
| 장점 | 복잡한 문제를 단계화 | 단일 풀이 오류를 줄임 |
| 단점 | 한 번 틀리면 끝 | 비용 증가 |

---

## 입문자가 꼭 기억해야 할 문장

**Self-Consistency는 CoT 풀이를 하나만 생성하지 않고 여러 개 샘플링한 뒤, 가장 많이 나온 최종 답을 선택해 reasoning 성능을 높이는 decoding 방법입니다.**

더 짧게 말하면:

> **Self-Consistency = 여러 번 생각하게 하고, 합의된 답을 고르는 방법**
