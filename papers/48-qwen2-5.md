# Qwen2.5 Technical Report

## 결론부터

이 논문의 결론은 이겁니다.

**Qwen2.5는 하나의 모델이 아니라, 0.5B부터 72B까지의 open-weight dense model, API용 MoE 모델인 Qwen2.5-Turbo/Plus, 그리고 Math·Coder·QwQ·멀티모달 모델의 기반이 되는 대규모 LLM 패밀리입니다.**

핵심은 **18T token 규모의 pre-training**, **1M+ supervised fine-tuning data**, **DPO와 GRPO를 포함한 multi-stage post-training**, **긴 context와 structured data/tool use 개선**입니다.

한 줄로 말하면:

> **Qwen2.5 = 18T token으로 학습하고, SFT+DPO+GRPO로 정렬한 다중 크기·다중 용도 LLM 패밀리**

---

## 이 논문을 한 문장으로 요약하면

**Qwen2.5 Technical Report는 “데이터 스케일링, 모델 크기 다양화, 긴 context, 수학·코딩·구조화 데이터, post-training alignment를 모두 강화한 중국 Alibaba Qwen 계열 LLM 패밀리 보고서”입니다.**

---

## 왜 Qwen2.5가 중요했나?

중요한 포인트는 네 가지입니다.

```text
1. 18T token pre-training으로 지식·수학·코딩 능력 강화
2. 0.5B~72B까지 다양한 open-weight 모델 크기 제공
3. SFT + DPO + GRPO 기반 post-training으로 instruction following과 preference alignment 강화
4. long context, structured data, JSON/table, tool use를 실용적으로 개선
```

---

## 핵심 개념 1: Open-weight dense model과 API용 MoE model

Open-weight dense 모델:

```text
Qwen2.5-0.5B
Qwen2.5-1.5B
Qwen2.5-3B
Qwen2.5-7B
Qwen2.5-14B
Qwen2.5-32B
Qwen2.5-72B
```

API용 proprietary MoE 모델:

```text
Qwen2.5-Turbo
Qwen2.5-Plus
```

---

## 핵심 개념 2: 모델 크기와 context

작은 0.5B·1.5B·3B 모델은 32K context를 지원하고, 7B 이상 open-weight 모델은 128K context와 8K generation length를 지원합니다.

Qwen2.5-Turbo는 최대 1M token context를 지원합니다.

---

## 핵심 개념 3: Dense Transformer architecture

Qwen2.5 dense model은 Qwen2와 유사한 decoder-only Transformer 구조를 유지합니다.

주요 요소:

```text
Grouped Query Attention, GQA
SwiGLU activation
RoPE positional encoding
QKV bias
RMSNorm
pre-normalization
```

GQA는 KV cache memory를 줄이고 decoding 효율을 높입니다.

---

## 핵심 개념 4: 151,643 vocabulary tokenizer

Qwen2.5는 byte-level BPE 기반 Qwen tokenizer를 사용합니다.

```text
regular token vocabulary: 151,643개
control token 확장
일부 tool functionality token 포함
```

---

## 핵심 개념 5: 18T token pre-training data

Qwen2.5의 가장 중요한 변화 중 하나는 pre-training data를 Qwen2의 7T에서 18T token으로 확장한 것입니다.

데이터 개선 포인트:

```text
더 좋은 data filtering
수학·코딩 데이터 강화
synthetic data 개선
data mixture balancing
```

---

## 핵심 개념 6: 수학·코딩 데이터의 중요성

Qwen2.5는 Qwen2.5-Math와 Qwen2.5-Coder의 데이터를 pre-training에 통합해 수학·코딩 능력을 강화했습니다.

```text
수학 문제
step-by-step reasoning
코드 문제
unit test
algorithmic code
multilingual code instruction
```

---

## 핵심 개념 7: Scaling law를 hyperparameter 선택에도 사용

Qwen2.5는 scaling law를 모델 크기뿐 아니라 learning rate와 batch size 같은 hyperparameter 선택에도 사용했습니다.

```text
모델 크기 N
데이터 크기 D
최적 batch size B
최적 learning rate μ
```

---

## 핵심 개념 8: Long-context pre-training

Qwen2.5는 먼저 4,096 token context로 학습한 뒤, 마지막 pre-training 단계에서 긴 context로 확장합니다.

일반 모델:

```text
4,096 → 32,768 token
YARN + DCA로 131,072 token 처리
```

Qwen2.5-Turbo:

```text
32K → 65K → 131K → 262K
inference 최대 1M token
```

---

## 핵심 개념 9: Post-training — SFT + Offline RL + Online RL

Qwen2.5 post-training은 세 축입니다.

```text
1. SFT: 1M+ high-quality examples
2. Offline RL: reasoning, factuality, instruction-following 등
3. Online RL: reward model 기반 GRPO
```

DPO와 GRPO가 모두 사용됩니다.

---

## 핵심 개념 10: SFT data coverage

Qwen2.5 SFT data는 단순 QA가 아니라 넓은 범위를 포함합니다.

```text
General instruction
Mathematics
Coding
Instruction-following
Structured data understanding
Logical reasoning
Cross-lingual transfer
Robust system instruction
Response filtering
```

---

## 핵심 개념 11: Data contamination 방지

대형 LLM 평가에서 data contamination은 중요합니다.

```text
평가 문제나 정답이 training data에 들어가 있으면
모델이 진짜로 푼 것이 아니라 본 것을 재생산할 수 있음
```

Qwen2.5는 n-gram/LCS matching으로 contaminated data를 제거했습니다.

## Qwen2.5와 Llama 3 비교

| 구분 | Llama 3 Herd | Qwen2.5 |
|---|---|---|
| 모델 크기 | 8B, 70B, 405B | 0.5B, 1.5B, 3B, 7B, 14B, 32B, 72B |
| 구조 | Dense Transformer | Open-weight dense, API MoE |
| Pre-training token | 약 15T+ | 18T |
| Context | 128K | 7B 이상 128K, Turbo 1M |
| Post-training | SFT, rejection sampling, DPO | SFT, DPO, GRPO, Offline/Online RL |
| 강점 | 405B dense flagship, safety stack | 작은 모델 라인업, 중국어·다국어, math/code, Turbo/Plus API |

---

## Qwen2.5의 장점

1. 크기 라인업이 촘촘합니다.
2. 18T token data scaling.
3. 수학·코딩 성능이 강합니다.
4. 긴 context와 structured data를 강화했습니다.
5. 한국어 benchmark를 포함한 다국어 평가가 있습니다.

---

## 한계와 주의점

1. 모델별 license가 다릅니다.
2. Qwen2.5-Turbo/Plus는 open-weight가 아니라 API용 proprietary MoE 모델입니다.
3. Long-context benchmark가 실제 long-document reasoning 전체를 대표하지는 않습니다.
4. reward model benchmark가 실제 RL 성능을 완전히 예측하지 못할 수 있습니다.
5. in-house benchmark는 외부 재현성이 제한될 수 있습니다.

---

## 입문자가 꼭 기억해야 할 문장

**Qwen2.5는 18T token pre-training과 SFT+DPO+GRPO post-training을 통해, 0.5B부터 72B까지 다양한 open-weight dense model과 API용 MoE 모델을 제공하고, 수학·코딩·다국어·long context·structured data·tool use 능력을 강화한 LLM 패밀리입니다.**

더 짧게 말하면:

> **Qwen2.5 = 데이터 스케일링과 실용 post-training을 강화한 다중 크기 LLM 패밀리**
