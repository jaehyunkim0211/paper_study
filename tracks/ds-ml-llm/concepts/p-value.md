# p-value

## 결론부터

**p-value는 “모형의 평가지표”라기보다는, 어떤 가설검정에서 관측된 결과가 귀무가설 아래에서 얼마나 보기 드문지를 나타내는 값**입니다.

가장 먼저 기존 이해를 보정하면 이렇게 됩니다.

> p-value가 0.05보다 작으면, “기존 가설”을 기각한다기보다 **귀무가설(null hypothesis, H₀)을 기각할 근거가 있다**고 말한다.

그리고 중요한 점은, **p-value는 모델이 얼마나 예측을 잘하는지를 평가하는 지표가 아닙니다.**  
예측 성능 평가지표는 RMSE, MAE, accuracy, AUC, F1-score 같은 것들이고, p-value는 주로 **통계적 가설검정**에서 쓰입니다.

---

## 1. p-value의 핵심 정의

**p-value는 “귀무가설이 참이라고 가정했을 때, 지금 관측한 결과 또는 그보다 더 극단적인 결과가 나올 확률”입니다.**

조금 더 풀면:

> “아무 효과가 없다고 가정했는데도, 지금처럼 차이가 크게 관측될 가능성이 얼마나 되는가?”

입니다.

예를 들어 A/B 테스트를 했다고 해봅시다.

| 그룹 | 구매전환율 |
|---|---:|
| 기존 페이지 A | 10% |
| 새 페이지 B | 13% |

여기서 우리가 궁금한 것은:

> “새 페이지 B가 진짜 더 좋은가?”

입니다.

그런데 통계검정에서는 보통 반대로 시작합니다.

### 귀무가설 H₀

> 새 페이지 B는 기존 페이지 A와 차이가 없다.

즉:

> 전환율 차이 3%p는 그냥 우연히 나온 것이다.

라고 일단 가정합니다.

그다음 p-value를 계산합니다.

만약 p-value가 0.03이라면 의미는 이겁니다.

> “실제로 A와 B의 차이가 없다고 가정했을 때, 지금처럼 3%p 이상의 차이가 관측될 확률은 약 3%다.”

그래서 우리는 이렇게 판단합니다.

> “차이가 없는데 이런 결과가 나올 확률이 3%밖에 안 된다면, 차이가 없다는 가정은 좀 이상하다. 귀무가설을 기각하자.”

---

## 2. 기존 이해에서 보강할 부분

기존 이해:

> “일반적으로 0.05보다 작은 경우, 유의미한 가설로 기존 가설을 기각한다.”

더 정확히는 이렇게 말하는 게 좋습니다.

> **p-value < 0.05이면, 유의수준 5%에서 귀무가설을 기각한다. 즉, 관측된 결과가 우연만으로 설명되기 어렵다고 판단한다.**

중요한 용어는 3개입니다.

| 용어 | 뜻 |
|---|---|
| **귀무가설 H₀** | 기본 가정. 보통 “차이 없음”, “효과 없음”, “관계 없음” |
| **대립가설 H₁** | 우리가 주장하고 싶은 가설. “차이 있음”, “효과 있음”, “관계 있음” |
| **유의수준 α** | 귀무가설을 기각하기로 정한 기준. 보통 0.05 |

즉, p-value는 혼자 존재하는 값이 아니라, 항상 **귀무가설과 유의수준**이 함께 있어야 해석됩니다.

---

## 3. p-value는 “귀무가설이 틀릴 확률”이 아니다

많은 사람들이 p-value를 이렇게 오해합니다.

> p-value = 귀무가설이 참일 확률

아닙니다.

p-value는 이것입니다.

> 귀무가설이 참이라고 가정했을 때, 지금 같은 데이터가 나올 확률

두 문장은 완전히 다릅니다.

### 잘못된 해석

> p-value가 0.03이니까 귀무가설이 참일 확률은 3%다.

이건 틀렸습니다.

### 올바른 해석

> 귀무가설이 참이라면, 지금처럼 극단적인 결과가 나올 확률이 3%다.

비유하면 이렇습니다.

동전을 10번 던졌는데 앞면이 10번 나왔습니다.

귀무가설은:

> 이 동전은 공정하다.

입니다.

공정한 동전이라면 앞면이 10번 연속 나올 확률은 매우 낮습니다.  
그래서 우리는 “동전이 공정하다는 가정이 의심스럽다”고 생각합니다.

하지만 이 말은:

> 동전이 공정할 확률이 정확히 몇 퍼센트다

라는 뜻은 아닙니다.

p-value는 **가설의 확률**이 아니라 **데이터의 희귀성**을 말합니다.

---

## 4. p-value < 0.05의 의미

보통 p-value가 0.05보다 작으면 “통계적으로 유의하다”고 말합니다.

하지만 0.05는 자연법칙이 아닙니다. 그냥 관습적으로 많이 쓰는 기준입니다.

| p-value | 일반적 해석 |
|---:|---|
| p < 0.001 | 매우 강한 통계적 근거 |
| p < 0.01 | 강한 통계적 근거 |
| p < 0.05 | 통계적으로 유의하다고 자주 판단 |
| p ≥ 0.05 | 귀무가설을 기각할 충분한 근거가 부족 |

중요한 표현은 이것입니다.

> p-value가 0.05보다 크다고 해서 귀무가설이 참이라는 뜻은 아니다.

예를 들어 p-value가 0.12라면:

> “차이가 없다”

가 아니라,

> “차이가 있다고 말할 만큼 충분한 증거가 없다”

입니다.

---

## 5. p-value는 효과의 크기를 말해주지 않는다

p-value는 **효과가 얼마나 큰지**를 말해주지 않습니다.

예를 들어 어떤 약이 혈압을 낮추는지 실험했다고 해봅시다.

| 결과 | 해석 |
|---|---|
| 평균 혈압 0.3mmHg 감소, p = 0.001 | 통계적으로 유의하지만 실질적 의미는 작을 수 있음 |
| 평균 혈압 8mmHg 감소, p = 0.08 | 통계적으로 유의하지 않지만 효과 크기는 커 보일 수 있음 |

왜 이런 일이 생기냐면 p-value는 다음 요소들의 영향을 받기 때문입니다.

| 요소 | p-value에 미치는 영향 |
|---|---|
| 효과 크기 | 효과가 클수록 p-value가 작아지기 쉬움 |
| 표본 크기 | 표본이 클수록 p-value가 작아지기 쉬움 |
| 데이터 변동성 | 변동성이 작을수록 p-value가 작아지기 쉬움 |
| 검정 방법 | t-test, z-test, chi-square test 등에 따라 달라짐 |

특히 표본 수가 매우 크면, 아주 작은 차이도 p-value가 작게 나올 수 있습니다.

예를 들어 사용자 1,000만 명의 데이터를 분석하면, 전환율이 10.000%에서 10.005%로 오른 것도 p-value는 매우 작게 나올 수 있습니다.

하지만 비즈니스적으로는 거의 의미가 없을 수 있습니다.

그래서 p-value만 보면 안 됩니다.

함께 봐야 하는 것들이 있습니다.

| 함께 볼 것 | 이유 |
|---|---|
| 효과 크기 effect size | 실제 차이가 얼마나 큰지 |
| 신뢰구간 confidence interval | 효과의 불확실성이 어느 정도인지 |
| 표본 크기 sample size | 결과가 표본 수에 의해 과도하게 유의해진 건 아닌지 |
| 실무적 의미 practical significance | 실제 의사결정에 도움이 되는지 |

## 6. p-value와 신뢰구간의 관계

p-value와 신뢰구간은 연결되어 있습니다.

예를 들어 평균 차이에 대한 95% 신뢰구간이 있다고 해봅시다.

| 평균 차이의 95% 신뢰구간 | 해석 |
|---|---|
| [1.2, 4.8] | 0을 포함하지 않음 → 보통 p < 0.05 |
| [-0.5, 3.1] | 0을 포함함 → 보통 p ≥ 0.05 |

차이가 없다는 값이 0이라고 할 때, 95% 신뢰구간에 0이 포함되지 않으면 보통 5% 유의수준에서 귀무가설을 기각합니다.

p-value는 “유의한가?”에 가까운 질문에 답하고, 신뢰구간은 “얼마나 차이가 날 수 있는가?”에 답합니다.

## 7. p-value가 쓰이는 대표적인 곳들

### 1. 평균 차이 검정: t-test

예를 들어:

> 남성과 여성의 평균 구매금액이 다른가?  
> 교육 전후의 시험 점수가 달라졌는가?  
> A 상품과 B 상품의 평균 만족도가 다른가?

이런 문제에서 t-test를 사용하고 p-value를 봅니다.

| 비교 | p-value | 해석 |
|---|---:|---|
| 교육 전 평균 점수 vs 교육 후 평균 점수 | 0.02 | 교육 전후 평균 차이가 통계적으로 유의함 |
| A 상품 만족도 vs B 상품 만족도 | 0.31 | 평균 차이가 있다고 보기에는 근거 부족 |

### 2. 비율 차이 검정: A/B 테스트

데이터사이언스 실무에서 매우 자주 쓰입니다.

| 지표 | A안 | B안 |
|---|---:|---:|
| 클릭률 | 4.1% | 4.5% |
| 구매전환율 | 1.8% | 2.1% |
| 이탈률 | 32% | 29% |

이런 지표 차이가 단순 우연인지, 실제 차이인지 판단할 때 p-value를 사용합니다.

### 3. 회귀분석에서 변수의 유의성 판단

회귀분석에서는 각 변수의 계수에 p-value가 붙습니다.

예를 들어 집값을 예측하는 회귀모형이 있다고 합시다.

```text
집값 = β₀ + β₁ × 면적 + β₂ × 역세권 여부 + β₃ × 건축연수 + 오차
```

회귀분석 결과가 다음과 같다고 해봅시다.

| 변수 | 계수 | p-value |
|---|---:|---:|
| 면적 | 500만 원 | 0.001 |
| 역세권 여부 | 3,000만 원 | 0.02 |
| 건축연수 | -200만 원 | 0.18 |

이때 해석은 다음과 같습니다.

| 변수 | 해석 |
|---|---|
| 면적 | 집값과 통계적으로 유의한 관계가 있어 보임 |
| 역세권 여부 | 집값과 통계적으로 유의한 관계가 있어 보임 |
| 건축연수 | 이 모형에서는 유의하다고 보기 어려움 |

하지만 회귀계수의 p-value는 다중공선성, 모형 형태, 누락 변수, 표본 크기, 이상치에 민감합니다.

즉, 회귀분석에서 p-value는:

> “이 변수는 타깃과 관련이 있는가?”

를 보는 데 도움이 되지만,

> “이 변수가 원인이다”

를 자동으로 보장하지는 않습니다.

### 4. 카이제곱 검정: 범주형 변수의 관계 확인

범주형 데이터에서는 chi-square test를 자주 씁니다.

예를 들어:

> 성별과 구매 여부가 관련이 있는가?  
> 지역과 선호 브랜드가 관련이 있는가?  
> 직업군과 상품 선택이 독립적인가?

귀무가설은 보통:

> 두 범주형 변수는 독립이다.

p-value가 0.01이면:

> 두 변수는 독립이라고 보기 어렵다.

라고 해석할 수 있습니다.

### 5. ANOVA: 여러 그룹 평균 비교

그룹이 2개일 때는 t-test를 많이 쓰고, 그룹이 3개 이상이면 ANOVA를 자주 씁니다.

ANOVA의 귀무가설은 보통:

> 모든 그룹의 평균은 같다.

입니다.

p-value가 0.03이면:

> 적어도 한 그룹의 평균은 다른 그룹들과 다르다고 볼 근거가 있다.

입니다.

하지만 ANOVA는 “어느 그룹이 다른지”까지 바로 알려주지는 않습니다. 그다음에는 post-hoc test, 예를 들면 Tukey test 같은 것을 사용합니다.

### 6. 모델 진단과 적합도 검정

p-value는 모델을 만들고 난 뒤의 진단에도 쓰입니다.

| 검정 | 질문 |
|---|---|
| Shapiro-Wilk test | 잔차가 정규분포를 따르는가? |
| Breusch-Pagan test | 잔차의 분산이 일정한가? |
| Durbin-Watson test | 잔차에 자기상관이 있는가? |
| Ljung-Box test | 시계열 잔차에 자기상관이 남아 있는가? |
| Goodness-of-fit test | 관측 분포가 이론 분포와 잘 맞는가? |

모델 진단에서는 p-value를 기계적으로 적용하면 안 됩니다. 표본이 크면 아주 작은 가정 위반도 유의하게 나오고, 표본이 작으면 큰 위반도 못 잡을 수 있습니다.

그래서 모델 진단에서는 p-value와 함께 그래프를 봅니다.

| 진단 대상 | 함께 볼 그래프 |
|---|---|
| 정규성 | Q-Q plot |
| 등분산성 | residual plot |
| 이상치 | leverage plot, Cook’s distance |
| 시계열 자기상관 | ACF/PACF plot |

## 8. p-value의 흔한 오해들

### 오해 1. p-value가 작으면 효과가 크다

아닙니다.

p-value가 작다는 것은 “우연으로 보기 어렵다”는 뜻이지, “효과가 크다”는 뜻은 아닙니다.

효과 크기는 coefficient, mean difference, odds ratio, Cohen’s d 같은 값을 봐야 합니다.

### 오해 2. p-value가 크면 효과가 없다

아닙니다.

p-value가 크다는 것은 “효과가 있다고 말할 증거가 부족하다”는 뜻입니다.

표본 수가 부족해서 못 잡았을 수도 있습니다.

### 오해 3. p-value가 0.049면 성공, 0.051이면 실패

실무적으로는 거의 같은 수준의 증거입니다.

0.05 기준 때문에 하나는 “유의”, 하나는 “비유의”로 나뉩니다.

그래서 p-value를 볼 때는 절단선 하나로만 판단하지 말고, 효과 크기와 신뢰구간을 함께 봐야 합니다.

### 오해 4. p-value가 낮으면 모델이 좋다

아닙니다.

회귀분석에서 어떤 변수의 p-value가 낮다고 해서 전체 모델의 예측 성능이 좋다는 뜻은 아닙니다.

예측 모델에서는 다음을 봐야 합니다.

| 문제 유형 | 예측 성능 지표 |
|---|---|
| 회귀 | RMSE, MAE, R² |
| 분류 | accuracy, precision, recall, F1, AUC |
| 랭킹 | NDCG, MAP, MRR |
| 확률예측 | log loss, Brier score |

### 오해 5. p-value가 유의하면 인과관계다

아닙니다.

p-value는 보통 “관계가 우연으로 보기 어려운가?”를 말해줄 뿐입니다.

인과관계를 주장하려면 더 많은 조건이 필요합니다.

| 필요한 것 | 이유 |
|---|---|
| 실험 설계 | 처치를 무작위로 배정해야 인과 해석이 쉬움 |
| 교란변수 통제 | 숨은 변수가 결과를 왜곡할 수 있음 |
| 시간 순서 | 원인이 결과보다 먼저 발생해야 함 |
| 도메인 지식 | 통계 결과만으로 인과를 단정하기 어려움 |

## 9. 다중검정 문제

**검정을 많이 하면, 우연히 p < 0.05가 나오는 결과가 생깁니다.**

예를 들어 실제로는 아무 효과가 없는 가설 100개를 검정한다고 해봅시다. 각 검정의 유의수준을 0.05로 두면, 평균적으로 약 5개는 우연히 유의하게 나올 수 있습니다.

이게 다중검정 문제입니다.

예를 들어 이런 경우에 발생합니다.

| 상황 | 문제 |
|---|---|
| 변수 100개 중 유의한 변수 찾기 | 우연히 유의한 변수가 나올 수 있음 |
| 고객 세그먼트 50개로 쪼개서 A/B 테스트 | 특정 세그먼트에서 우연히 유의한 차이 발생 |
| 여러 지표를 동시에 비교 | 클릭률, 전환율, 체류시간 중 하나는 우연히 유의할 수 있음 |
| 논문에서 많은 분석 중 유의한 것만 보고 | p-hacking 위험 |

그래서 다중검정에서는 보정이 필요할 수 있습니다.

| 방법 | 설명 |
|---|---|
| Bonferroni correction | 기준을 매우 엄격하게 조정 |
| Holm correction | Bonferroni보다 덜 보수적인 순차 보정 |
| Benjamini-Hochberg | false discovery rate를 통제 |
| 사전 등록 pre-registration | 분석 계획을 미리 정해 p-hacking 방지 |

---

## 10. p-hacking이란?

p-hacking은 쉽게 말해:

> p-value가 0.05보다 작게 나올 때까지 분석을 이것저것 바꿔보는 행위

입니다.

예를 들면:

| p-hacking 예시 |
|---|
| 유의하지 않은 변수를 빼고 다시 분석 |
| 표본 일부를 제외하고 다시 분석 |
| 여러 지표 중 유의한 지표만 보고 |
| 여러 모델 중 p-value가 가장 작은 결과만 선택 |
| 데이터 수집을 중간중간 멈췄다 재개했다 반복 |

좋은 분석에서는 다음이 중요합니다.

| 원칙 | 이유 |
|---|---|
| 분석 계획을 미리 정하기 | 결과 보고 분석 방법을 바꾸지 않기 위해 |
| 주요 지표를 사전에 정하기 | 여러 지표 중 유리한 것만 고르는 것을 방지 |
| 전체 검정 수를 보고하기 | 다중검정 문제를 확인하기 위해 |
| 효과 크기와 신뢰구간도 함께 제시 | p-value 중심 해석을 줄이기 위해 |
| 재현성 확인 | 한 번의 유의한 결과에 과도하게 의존하지 않기 위해 |

---

## 11. 데이터사이언스에서 p-value를 어떻게 써야 하나?

### 좋은 사용

| 사용 방식 | 예시 |
|---|---|
| 실험 결과 판단 | A/B 테스트에서 전환율 차이가 우연인지 판단 |
| 변수 관계 탐색 | 특정 변수가 결과와 관련이 있는지 확인 |
| 회귀분석 해석 | 변수의 계수가 0과 다른지 검정 |
| 모델 가정 점검 | 잔차 정규성, 등분산성, 독립성 확인 |
| 연구 결과 보고 | 효과와 불확실성을 통계적으로 설명 |

### 조심해야 하는 사용

| 사용 방식 | 왜 조심해야 하나 |
|---|---|
| feature selection에 p-value만 사용 | 예측 성능과 직접 연결되지 않을 수 있음 |
| p < 0.05만 보고 결론 | 효과 크기와 실무적 의미를 놓칠 수 있음 |
| 여러 검정 중 유의한 것만 보고 | 다중검정, p-hacking 위험 |
| 상관관계를 인과관계로 해석 | p-value는 인과를 보장하지 않음 |
| 모델 성능 지표처럼 사용 | 예측력 평가가 아님 |

## 12. p-value를 볼 때의 실전 체크리스트

### 1단계: 귀무가설이 무엇인가?

p-value는 항상 귀무가설에 대한 값입니다.

> 무엇이 “없다”고 가정했는가?

| 분석 | 귀무가설 |
|---|---|
| A/B 테스트 | A와 B의 전환율은 같다 |
| 회귀분석 | 이 변수의 계수는 0이다 |
| 카이제곱 검정 | 두 범주형 변수는 독립이다 |
| ANOVA | 모든 그룹의 평균은 같다 |

### 2단계: p-value가 얼마나 작은가?

| p-value | 느낌 |
|---:|---|
| 0.8 | 귀무가설과 매우 잘 양립 |
| 0.2 | 유의하다고 보기 어려움 |
| 0.049 | 기준상 유의하지만 약한 근거 |
| 0.001 | 강한 근거 |
| 0.000001 | 매우 강한 근거일 수 있으나 표본 크기도 봐야 함 |

### 3단계: 효과 크기는 얼마인가?

p-value보다 더 중요한 경우가 많습니다.

> 차이가 있긴 한데, 그 차이가 의미 있는 크기인가?

### 4단계: 신뢰구간은 어떤가?

신뢰구간은 효과의 범위를 보여줍니다.

### 5단계: 여러 번 검정했는가?

검정을 많이 했다면 p-value 기준을 그대로 쓰면 위험합니다.

> 지표를 몇 개 봤는가?  
> 세그먼트를 몇 개 나눴는가?  
> 모델을 몇 개 비교했는가?  
> 유의한 것만 골라 보고 있지는 않은가?

## 13. 아주 짧은 예시로 정리

온라인 강의 플랫폼에서 새 추천 알고리즘을 도입했다고 합시다.

| 그룹 | 수강 신청률 |
|---|---:|
| 기존 추천 | 8.0% |
| 새 추천 | 8.8% |

검정 결과:

> p-value = 0.03

올바른 해석:

> 기존 추천과 새 추천의 실제 수강 신청률이 같다고 가정하면, 지금처럼 0.8%p 이상의 차이가 관측될 확률은 약 3%다. 따라서 5% 유의수준에서는 “두 추천 방식의 신청률이 같다”는 귀무가설을 기각할 수 있다.

하지만 여기서 바로 끝내면 안 됩니다.

| 질문 | 이유 |
|---|---|
| 0.8%p 상승이 비즈니스적으로 큰가? | 실무적 의미 확인 |
| 신뢰구간은 얼마인가? | 불확실성 확인 |
| 표본 수는 충분한가? | 검정력 확인 |
| 다른 지표는 악화되지 않았는가? | 부작용 확인 |
| 여러 지표 중 하나만 유의한 것은 아닌가? | 다중검정 확인 |

---

## 14. p-value를 한 문장으로 정확히 말하는 법

앞으로 p-value를 설명할 때는 이렇게 말하면 됩니다.

> **p-value는 귀무가설이 참이라고 가정했을 때, 현재 관측한 결과 또는 그보다 더 극단적인 결과가 나올 확률이다. p-value가 작을수록 귀무가설과 관측 데이터가 잘 맞지 않는다는 뜻이며, 보통 0.05보다 작으면 통계적으로 유의하다고 판단한다. 다만 p-value는 효과의 크기, 실무적 중요성, 인과관계, 모델 예측력을 직접 말해주지는 않는다.**

## 최종 요약

| 기존 이해 | 보강된 이해 |
|---|---|
| p-value는 모형의 평가지표다 | 정확히는 가설검정의 통계적 증거 지표다 |
| 0.05보다 작으면 유의미하다 | 5% 유의수준에서 귀무가설을 기각할 수 있다 |
| 기존 가설을 기각한다 | 보통 “차이 없음/효과 없음”이라는 귀무가설을 기각한다 |
| p-value가 작으면 좋은 결과다 | 작은 p-value는 우연으로 보기 어렵다는 뜻이지, 효과가 크거나 모델이 좋다는 뜻은 아니다 |
| p-value만 보면 된다 | 효과 크기, 신뢰구간, 표본 수, 다중검정, 실무적 의미를 함께 봐야 한다 |

데이터사이언스 관점에서 가장 중요한 정리는 이것입니다.

**p-value는 “이 차이가 우연으로 보기 어려운가?”를 판단하는 도구이고, 모델의 예측 성능이나 비즈니스 가치를 평가하는 도구는 아니다.**
