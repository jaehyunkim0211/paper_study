# SHAP 쉬운 예시: Linear Model과 Random Forest에서 어떻게 적용하나?

## 결론부터

**SHAP은 기존에 학습된 모델을 바꾸거나 재학습하지 않습니다.**

이미 학습된 모델이 있을 때,

```text
이 모델의 예측값을 feature별 기여도로 쪼개서 설명하는 후처리 계산
```

을 하는 것입니다.

즉 흐름은 이렇습니다.

```text
1. 모델을 학습한다.
2. 어떤 샘플 하나에 대해 예측값을 구한다.
3. SHAP explainer를 사용해 그 예측값을 feature별 기여도로 분해한다.
```

SHAP은 모델을 새로 학습하는 게 아니라, **이미 학습된 모델의 예측 함수 f(x)를 분석하는 방법**입니다.

---

## 1. 가장 쉬운 예: Linear Model에서 SHAP

선형회귀 모델이 있다고 해봅시다.

학생의 시험 점수를 예측하는 모델입니다.

```text
예측 점수 = 50 + 5 × 공부시간 - 10 × 결석횟수
```

여기서 모델은 이미 학습되어 있다고 합시다.

| 항목 | 값 |
|---|---:|
| 절편 | 50 |
| 공부시간 계수 | +5 |
| 결석횟수 계수 | -10 |

즉, 이 모델은 이렇게 말합니다.

```text
공부시간이 1시간 늘면 점수 +5점
결석이 1회 늘면 점수 -10점
```

---

## 2. SHAP의 기준값: base value

SHAP은 보통 **평균적인 예측값**에서 시작합니다.

전체 학생의 평균적인 feature 값이 다음과 같다고 해봅시다.

| Feature | 평균값 |
|---|---:|
| 공부시간 | 4시간 |
| 결석횟수 | 1회 |

그러면 평균적인 학생의 예측 점수는:

```text
base value = 50 + 5 × 4 - 10 × 1
           = 50 + 20 - 10
           = 60점
```

즉, SHAP의 출발점은:

```text
평균적인 학생은 60점으로 예측된다.
```

입니다.

---

## 3. 특정 학생 한 명을 설명해보자

이제 어떤 학생 A가 있습니다.

| Feature | 학생 A |
|---|---:|
| 공부시간 | 7시간 |
| 결석횟수 | 0회 |

모델의 예측값은:

```text
예측 점수 = 50 + 5 × 7 - 10 × 0
         = 50 + 35
         = 85점
```

모델은 이 학생을 **85점**으로 예측했습니다.

이제 SHAP은 이렇게 묻습니다.

> 평균적인 예측 60점에서 출발해서,  
> 왜 이 학생은 85점이 되었을까?

---

## 4. SHAP value 계산

### 공부시간의 기여

평균 공부시간은 4시간입니다.

학생 A는 7시간 공부했습니다.

```text
차이 = 7 - 4 = 3시간
```

공부시간 계수는 +5입니다.

```text
공부시간 SHAP value = 5 × 3 = +15점
```

해석:

> 학생 A는 평균보다 공부를 3시간 더 했기 때문에 예측 점수가 15점 올라갔다.

### 결석횟수의 기여

평균 결석횟수는 1회입니다.

학생 A는 0회 결석했습니다.

```text
차이 = 0 - 1 = -1회
```

결석횟수 계수는 -10입니다.

```text
결석횟수 SHAP value = -10 × (-1) = +10점
```

해석:

> 학생 A는 평균보다 결석을 1회 덜 했기 때문에 예측 점수가 10점 올라갔다.

---

## 5. SHAP 설명 완성

이제 base value와 SHAP value를 더합니다.

```text
최종 예측값
= base value
+ 공부시간 SHAP value
+ 결석횟수 SHAP value

= 60 + 15 + 10
= 85점
```

표로 보면 이렇게 됩니다.

| 항목 | 기여 |
|---|---:|
| 평균적인 예측값, base value | 60점 |
| 공부시간 7시간 | +15점 |
| 결석 0회 | +10점 |
| 최종 예측값 | 85점 |

이게 SHAP의 기본 구조입니다.

```text
모델 예측값 = base value + feature별 SHAP value의 합
```

---

## 6. 이 예시에서 SHAP이 말해주는 것

이 학생의 점수가 85점으로 예측된 이유는:

```text
평균적인 학생보다 공부시간이 많아서 +15점
평균적인 학생보다 결석이 적어서 +10점
그래서 평균 예측 60점에서 85점으로 올라감
```

입니다.

즉, SHAP은 단순히:

> 공부시간이 중요하다

라고 말하는 것이 아닙니다.

더 구체적으로:

> **이 학생의 이번 예측에서 공부시간은 +15점, 결석횟수는 +10점 기여했다**

라고 말합니다.

---

## 7. Linear model에서는 SHAP이 특히 직관적이다

선형모델에서는 가장 단순한 경우 SHAP이 거의 이렇게 계산됩니다.

```text
feature j의 SHAP value
= 계수 × (해당 샘플의 feature 값 - 평균 feature 값)
```

즉:

```text
SHAP value = coefficient × deviation from average
```

예를 들어:

```text
공부시간 SHAP = 5 × (7 - 4) = +15
결석횟수 SHAP = -10 × (0 - 1) = +10
```

그래서 선형모델에서는 SHAP이 회귀계수 해석과 상당히 잘 연결됩니다.

다만 feature들 사이에 상관관계가 강하면 SHAP 계산 방식에 따라 기여도 배분이 조금 더 복잡해질 수 있습니다. 입문 단계에서는 위 예시처럼 이해하면 충분합니다.

---

## 8. Random Forest에서는 어떻게 적용하나?

Random Forest도 원리는 같습니다.

다만 선형모델처럼 간단한 계수 공식이 없습니다.

Random Forest는 여러 tree의 평균으로 예측합니다.

```text
Random Forest 예측값
= tree 1 예측
+ tree 2 예측
+ ...
+ tree N 예측의 평균
```

그래서 SHAP은 이렇게 합니다.

```text
평균 예측값에서 출발해서,
각 feature가 tree들의 예측을 얼마나 올리거나 내렸는지 계산한다.
```

예를 들어 고객 이탈 Random Forest 모델이 있다고 해봅시다.

전체 고객의 평균 이탈 예측값이:

```text
base value = 0.20
```

즉, 평균적으로 이탈 확률 20%라고 합시다.

어떤 고객 A에 대해 Random Forest가:

```text
예측 이탈 확률 = 0.70
```

이라고 했습니다.

SHAP은 이 0.20에서 0.70까지의 차이 0.50을 feature별로 나눕니다.

| Feature | SHAP value | 해석 |
|---|---:|---|
| 가입기간 1개월 | +0.18 | 이탈 확률을 올림 |
| 최근 사용량 급감 | +0.22 | 이탈 확률을 올림 |
| 불만 접수 2회 | +0.15 | 이탈 확률을 올림 |
| 할인 적용 중 | -0.05 | 이탈 확률을 낮춤 |
| base value | 0.20 | 평균 이탈 확률 |
| 최종 예측 | 0.70 | 합산 결과 |

합치면:

```text
0.20 + 0.18 + 0.22 + 0.15 - 0.05 = 0.70
```

이런 식입니다.

---

## 9. Random Forest에서 SHAP은 재학습을 하나?

아니요.

**Random Forest 모델을 다시 학습하지 않습니다.**

이미 학습된 Random Forest가 있으면, SHAP은 그 tree 구조를 사용합니다.

개념적으로는 이런 질문을 합니다.

> 가입기간 feature를 알 때와 모를 때 예측이 얼마나 달라지는가?  
> 최근 사용량 feature를 알 때와 모를 때 예측이 얼마나 달라지는가?  
> 불만 접수 feature를 알 때와 모를 때 예측이 얼마나 달라지는가?

그런데 feature를 하나씩 추가하는 순서에 따라 기여도가 달라질 수 있습니다.

예를 들어:

```text
가입기간을 먼저 알고 최근 사용량을 나중에 아는 경우
최근 사용량을 먼저 알고 가입기간을 나중에 아는 경우
```

두 경우의 기여도가 다를 수 있습니다.

그래서 SHAP은 이론적으로는 **가능한 feature 추가 순서를 모두 고려해서 평균 기여도를 계산**합니다.

Random Forest나 XGBoost, LightGBM 같은 tree 모델에서는 이 계산을 효율적으로 해주는 **Tree SHAP** 알고리즘을 씁니다.

---

## 10. “feature를 모른다”는 게 무슨 뜻인가?

SHAP에서 중요한 직관입니다.

SHAP은 feature의 기여도를 계산할 때 이런 상태를 상상합니다.

```text
이 feature를 알고 있을 때의 예측
vs
이 feature를 모를 때의 평균적인 예측
```

예를 들어 고객 A의 `최근 사용량 급감`이라는 정보를 모른다고 해봅시다.

그러면 모델은 그 feature를 특정 값으로 쓰지 못하고, background data를 기준으로 평균적인 가능성을 고려합니다.

즉, SHAP에서 feature를 “뺀다”는 것은 보통:

```text
컬럼을 삭제하고 모델을 다시 학습한다
```

가 아닙니다.

대신:

```text
이미 학습된 모델은 그대로 두고,
그 feature 값을 모른다고 가정했을 때의 예측 기대값을 계산한다
```

에 가깝습니다.

---

## 11. 실제 코드 흐름은 이렇게 된다

개념적으로는 이런 순서입니다.

```python
# 1. 모델 학습
model.fit(X_train, y_train)

# 2. SHAP explainer 생성
explainer = shap.Explainer(model, X_background)

# 3. 특정 데이터에 대한 SHAP value 계산
shap_values = explainer(X_test_one)
```

여기서 중요한 것은:

```text
model.fit()은 한 번만 한다.
SHAP은 학습된 model을 받아서 설명값을 계산한다.
```

입니다.

즉, SHAP은 모델 학습 과정에 들어가는 게 아니라 보통은 **모델 학습 이후에 붙는 설명 단계**입니다.

---

## 12. 그럼 SHAP을 구하려면 추가로 필요한 것은?

보통 두 가지가 필요합니다.

### 1. 이미 학습된 모델

예:

```text
Linear Regression
Random Forest
XGBoost
LightGBM
Neural Network
```

### 2. background data

background data는 기준이 되는 데이터입니다.

예를 들어:

```text
평균적인 학생
평균적인 고객
평균적인 거래
```

를 정의하는 데 쓰입니다.

Linear model 예시에서는 background data의 평균값이:

```text
공부시간 평균 4시간
결석횟수 평균 1회
```

였습니다.

그래서 base value가 60점이었습니다.

Random Forest에서도 background data를 기준으로:

```text
평균 예측값, feature를 모를 때의 기대 예측값
```

을 계산합니다.

---

## 13. SHAP은 모델에 뭔가를 “추가 학습”하는 게 아니다

질문에 직접 답하면:

> 기존에 학습된 모델에 추가로 뭔가를 해서 SHAP을 구하는 거야?

정확히는:

**추가 학습을 하는 것은 아니고, 기존 모델을 대상으로 설명값을 계산하는 explainer를 붙이는 것입니다.**

비유하면 이렇습니다.

| 과정 | 비유 |
|---|---|
| 모델 학습 | 시험 채점기를 만든다 |
| 모델 예측 | 어떤 학생의 점수를 예측한다 |
| SHAP 계산 | 왜 그 점수가 나왔는지 항목별 채점 근거를 분석한다 |

SHAP은 채점기를 새로 만드는 게 아닙니다.

이미 만들어진 채점기를 보고:

```text
공부시간 때문에 +15점
결석이 적어서 +10점
```

처럼 설명을 붙이는 것입니다.

---

## 14. 최종 정리

SHAP 적용은 이렇게 이해하면 됩니다.

```text
기존 학습 모델 f(x)가 있다.
특정 샘플 x에 대해 예측값 f(x)를 구한다.
SHAP은 f(x)를 base value + feature별 기여도 합으로 분해한다.
모델은 재학습하지 않는다.
```

선형모델에서는 가장 단순하게:

```text
SHAP value = 계수 × (feature 값 - 평균 feature 값)
```

처럼 이해할 수 있습니다.

Random Forest에서는 계수 공식은 없지만, 원리는 같습니다.

```text
각 feature를 알 때와 모를 때 예측이 얼마나 달라지는지,
tree 구조를 이용해서 feature별 기여도로 나눠 계산한다.
```

가장 중요한 문장은 이겁니다.

**SHAP은 모델을 새로 학습하는 방법이 아니라, 이미 학습된 모델의 개별 예측값을 “평균 예측값 + feature별 기여도”로 분해하는 설명 방법입니다.**
