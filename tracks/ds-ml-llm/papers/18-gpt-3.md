# Language Models are Few-Shot Learners — GPT-3

## 결론부터

이 논문의 결론은 이겁니다.

**언어 모델을 충분히 크게 만들고 대규모 텍스트로 학습하면, 새로운 문제를 풀 때 모델의 weight를 다시 학습하지 않아도, 프롬프트 안에 설명과 예시 몇 개만 넣어 문제를 풀 수 있다.**

이게 GPT-3 논문의 핵심입니다.

한 줄로 말하면:

> **GPT-3는 “fine-tuning 없이도, 프롬프트에 적힌 지시문과 예시만 보고 새로운 task를 수행할 수 있다”는 가능성을 보여준 논문입니다.**

Brown et al.은 GPT-3를 **175B parameter를 가진 autoregressive language model**로 학습했고, 각 task에서 gradient update나 fine-tuning 없이 자연어 지시문과 few-shot demonstration만 텍스트로 제공해 평가했습니다. 논문은 모델 규모를 키우면 task-agnostic few-shot 성능이 크게 좋아진다고 주장합니다.

---

## 이 논문을 한 문장으로 요약하면

**GPT-3는 “모델을 task별로 다시 학습시키는 방식”에서 “모델에게 텍스트로 과제를 설명하고 예시를 보여주는 방식”으로 넘어가는 중요한 논문입니다.**

BERT와 비교하면 차이가 분명합니다.

| 모델 | 핵심 방식 |
|---|---|
| **BERT** | `[MASK]` 단어 맞히기로 양방향 encoder를 사전학습하고, task별로 fine-tuning |
| **GPT-3** | 다음 token 예측으로 autoregressive language model을 사전학습하고, task별 fine-tuning 없이 prompt로 수행 |

BERT는 보통 이런 흐름입니다.

```text
pre-training
→ task-specific labeled data로 fine-tuning
→ task 수행
```

GPT-3 논문이 강조한 흐름은 이겁니다.

```text
pre-training
→ prompt에 task 설명과 예시 제공
→ 바로 task 수행
```

---

## 왜 이 논문이 중요한가?

BERT 논문은 “대규모 사전학습 후 fine-tuning”의 힘을 보여줬습니다.

GPT-3 논문은 그다음 질문을 던집니다.

> 꼭 fine-tuning을 해야 할까? 모델이 충분히 크면, 그냥 예시 몇 개를 prompt에 넣는 것만으로도 task를 이해할 수 있지 않을까?

예를 들어 감성분석을 한다고 해봅시다.

BERT식 접근은 보통 이렇습니다.

```text
감성분석 labeled dataset 준비
→ BERT fine-tuning
→ 새 문장 분류
```

GPT-3식 접근은 이런 식입니다.

```text
다음 문장의 감성을 긍정 또는 부정으로 분류하세요.

문장: 이 영화는 정말 재미있었다.
감성: 긍정

문장: 음식이 차갑고 서비스도 별로였다.
감성: 부정

문장: 배우들의 연기가 훌륭했다.
감성:
```

모델은 마지막에 올 단어를 예측합니다.

```text
긍정
```

여기서 중요한 점은 **모델 weight를 업데이트하지 않는다는 것**입니다.

## 핵심 개념 1: Autoregressive Language Model

### 결론부터

**GPT-3는 앞의 token들을 보고 다음 token을 예측하도록 학습된 모델입니다.**

예를 들어 문장이 있습니다.

```text
나는 오늘 카페에서 커피를 마셨다
```

GPT식 학습은 이런 문제를 계속 푸는 것입니다.

```text
나는 → 오늘
나는 오늘 → 카페에서
나는 오늘 카페에서 → 커피를
나는 오늘 카페에서 커피를 → 마셨다
```

즉, 기본 학습 목표는 단순합니다.

> **지금까지의 텍스트를 보고 다음 token을 맞혀라.**

BERT는 중간 token을 가리고 양쪽 문맥으로 맞히는 MLM 방식이었지만, GPT-3는 왼쪽에서 오른쪽으로 다음 token을 예측하는 autoregressive 방식입니다.

---

## GPT-3는 왜 decoder 쪽에 가까운가?

Transformer 원 논문에는 encoder와 decoder가 있었습니다.

BERT는 encoder 쪽에 가깝습니다.

```text
문장 전체를 양방향으로 봄
→ 각 token의 문맥 표현 생성
```

GPT-3는 decoder 쪽에 가깝습니다.

```text
이전 token들만 봄
→ 다음 token 예측
```

예를 들어:

```text
나는 오늘 카페에서
```

여기까지만 보고 다음 token을 예측합니다.

```text
커피를
```

미래 token은 볼 수 없습니다.

이 점이 BERT와 GPT 계열의 가장 큰 차이입니다.

```text
BERT: 양쪽 문맥을 봄
GPT: 이전 문맥만 보고 다음 token 생성
```

## 핵심 개념 2: Zero-shot, One-shot, Few-shot

GPT-3 논문의 핵심은 이 세 가지 평가 방식입니다.

### 1. Zero-shot

**예시 없이 지시문만 주는 방식입니다.**

```text
다음 문장을 영어로 번역하세요.

문장: 나는 학교에 간다.
번역:
```

모델은 바로 답해야 합니다.

```text
I go to school.
```

### 2. One-shot

**예시 하나를 주고 문제를 푸는 방식입니다.**

```text
다음 문장을 영어로 번역하세요.

문장: 나는 밥을 먹는다.
번역: I eat rice.

문장: 나는 학교에 간다.
번역:
```

모델은 예시 하나를 보고 형식을 이해합니다.

```text
I go to school.
```

### 3. Few-shot

**예시 여러 개를 prompt 안에 넣고 문제를 푸는 방식입니다.**

```text
다음 문장을 영어로 번역하세요.

문장: 나는 밥을 먹는다.
번역: I eat rice.

문장: 그녀는 책을 읽는다.
번역: She reads a book.

문장: 우리는 음악을 듣는다.
번역: We listen to music.

문장: 나는 학교에 간다.
번역:
```

모델은 여러 예시를 보고 task의 패턴을 파악합니다.

```text
I go to school.
```

---

## 여기서 “learning”은 진짜 학습인가?

이 부분이 아주 중요합니다.

논문 제목은 **Language Models are Few-Shot Learners**입니다.

그런데 여기서 “learners”라고 해서 GPT-3가 few-shot 예시를 보고 **weight를 업데이트한다**는 뜻은 아닙니다.

정확히는:

```text
prompt 안의 예시를 보고,
현재 문맥 안에서 task 패턴을 파악한다.
하지만 모델 parameter는 바뀌지 않는다.
```

즉, 이것은 **in-context learning**입니다.

| 구분 | weight update 있음? | 예시 |
|---|---|---|
| Fine-tuning | 있음 | BERT를 감성분석 데이터로 추가 학습 |
| Few-shot prompting | 없음 | prompt 안에 감성분석 예시 몇 개 제공 |
| In-context learning | 없음 | 문맥 안에서 패턴을 파악해 답변 |

## 핵심 개념 3: Prompt가 task specification이 된다

BERT에서는 task를 수행하려면 보통 output head를 붙이고 fine-tuning했습니다.

예를 들어 감성분석이면:

```text
BERT + classification head
```

질의응답이면:

```text
BERT + span prediction head
```

GPT-3에서는 task를 텍스트로 설명합니다.

```text
다음 리뷰가 긍정인지 부정인지 판단하세요.
```

그리고 예시를 보여줍니다.

```text
리뷰: 정말 훌륭한 영화였다.
답: 긍정

리뷰: 지루하고 길기만 했다.
답: 부정
```

마지막 문제를 줍니다.

```text
리뷰: 배우들의 연기가 인상적이었다.
답:
```

GPT-3는 다음 token을 생성합니다.

```text
긍정
```

즉, prompt가 곧 task 설명서가 됩니다.

이게 GPT-3 이후 LLM 사용 방식의 핵심이 됩니다.

```text
모델 구조를 바꾸지 않고,
입력 텍스트를 바꿔서 task를 바꾼다.
```

---

## 쉬운 예시 1: 분류 문제

감성분석 예시를 봅시다.

```text
다음 리뷰를 긍정 또는 부정으로 분류하세요.

리뷰: 음식이 맛있고 직원이 친절했다.
분류: 긍정

리뷰: 배송이 늦고 제품도 망가져 있었다.
분류: 부정

리뷰: 가격 대비 품질이 아주 좋았다.
분류:
```

GPT-3는 다음에 올 단어를 예측합니다.

```text
긍정
```

겉보기에는 분류 모델처럼 작동하지만, 실제로는 다음 token 예측입니다.

```text
"분류:" 다음에 올 token을 생성하는 것
```

---

## 쉬운 예시 2: 번역 문제

```text
한국어를 영어로 번역하세요.

한국어: 나는 밥을 먹었다.
영어: I ate rice.

한국어: 그녀는 책을 읽고 있다.
영어: She is reading a book.

한국어: 우리는 영화를 봤다.
영어:
```

GPT-3는 패턴을 보고 다음에 올 텍스트를 생성합니다.

```text
We watched a movie.
```

번역 전용 모델로 fine-tuning하지 않아도, prompt를 통해 번역 task를 수행하는 방식입니다.

## 핵심 개념 4: Scaling

### 결론부터

**GPT-3 논문의 핵심 메시지 중 하나는 모델을 크게 만들수록 few-shot 능력이 좋아진다는 것입니다.**

논문은 125M parameter부터 175B parameter까지 여러 크기의 모델을 학습해 비교했습니다. 가장 큰 모델이 GPT-3 175B입니다.

| 모델 | parameter 수 |
|---|---:|
| GPT-3 Small | 125M |
| GPT-3 Medium | 350M |
| GPT-3 XL | 1.3B |
| GPT-3 13B | 13B |
| GPT-3 | 175B |

여기서 중요한 건 단순히 “큰 모델이 더 좋다”가 아닙니다.

논문이 흥미롭게 본 것은:

> 모델이 커질수록, prompt 안의 예시를 활용하는 능력도 더 좋아진다.

입니다.

---

## 핵심 개념 5: 학습 데이터

GPT-3는 대량의 텍스트로 학습되었습니다.

주요 학습 데이터는 다음과 같습니다.

| 데이터 | 설명 |
|---|---|
| Filtered Common Crawl | 웹 크롤링 텍스트를 품질 필터링 |
| WebText2 | 웹 링크 기반 고품질 텍스트 |
| Books1, Books2 | 책 데이터 |
| Wikipedia | 영어 위키피디아 |

여기서 BERT와 비교하면 차이가 있습니다.

BERT는 MLM, NSP 같은 사전학습 task를 만들었습니다.

GPT-3는 더 단순합니다.

```text
대규모 텍스트를 읽고 다음 token을 예측한다.
```

즉, 별도의 `[MASK]`를 만들지 않습니다.

---

## BERT와 GPT-3를 다시 비교하면

| 구분 | BERT | GPT-3 |
|---|---|---|
| Transformer 구조 | Encoder-only | Decoder-like autoregressive model |
| 보는 방향 | 양방향 | 왼쪽 → 오른쪽 |
| 사전학습 목표 | Masked token 예측, NSP | 다음 token 예측 |
| 주요 강점 | 언어 이해, 분류, 추출 | 생성, prompt 기반 task 수행 |
| downstream 사용 | fine-tuning 중심 | zero/one/few-shot prompting 중심 |
| task 변경 방식 | head 추가 + fine-tuning | prompt 변경 |

가장 쉽게 말하면:

```text
BERT는 빈칸 맞히기로 문장을 이해하는 모델
GPT-3는 다음 단어 맞히기로 글을 이어 쓰는 모델
```

그런데 GPT-3는 규모가 매우 커지면서, 단순히 글을 이어 쓰는 것뿐 아니라 prompt 안의 예시를 보고 다양한 task를 수행할 수 있음을 보여줬습니다.

## 핵심 개념 6: Fine-tuning과 In-context Learning의 차이

## Fine-tuning

모델 weight를 실제로 업데이트합니다.

```text
데이터 1만 개 준비
→ loss 계산
→ backpropagation
→ weight update
→ task-specific model 생성
```

장점:

```text
특정 task에서 성능이 좋을 수 있음
```

단점:

```text
task마다 데이터와 학습이 필요함
overfitting 가능성 있음
배포와 관리가 복잡해질 수 있음
```

## In-context Learning

모델 weight는 그대로 둡니다.

```text
prompt에 task 설명과 예시 입력
→ 모델이 다음 token 생성
```

장점:

```text
별도 학습 없이 빠르게 task를 바꿀 수 있음
데이터가 적어도 시도 가능
```

단점:

```text
prompt 설계에 민감함
context window에 들어가는 예시 수가 제한됨
fine-tuned model보다 약할 수 있음
```

## GPT-3의 한계

### 1. 여전히 추론과 일관성에 약점이 있다

GPT-3는 텍스트 생성 품질이 높지만, 긴 글에서는 반복, 모순, coherence 저하, 엉뚱한 문장이나 문단을 생성할 수 있습니다.

쉽게 말하면:

```text
그럴듯한 문장을 잘 만들지만,
항상 정확하게 이해하거나 추론하는 것은 아니다.
```

### 2. 양방향 모델이 아니다

GPT-3는 autoregressive model입니다.

즉, 이전 token을 보고 다음 token을 예측합니다.

그래서 bidirectionality가 유리한 task에서는 BERT식 모델이 유리할 수 있습니다.

### 3. 데이터 오염 문제가 있다

GPT-3는 웹 기반 대규모 corpus로 학습했습니다. 이 경우 benchmark test set의 일부가 training data에 섞여 있을 수 있습니다.

이 부분은 이전에 배운 **Model Evaluation** 논문과 연결됩니다.

```text
train과 test가 섞이면 성능 평가가 오염된다.
```

### 4. 편향과 오용 위험이 있다

대규모 웹 텍스트를 학습한 모델은 사회적 편향과 유해한 사용 가능성을 가질 수 있습니다.

## BERT에서 GPT-3로 넘어가는 의미

이전까지 NLP는 점점 이런 흐름이었습니다.

```text
word embedding
→ contextual embedding
→ pre-training + fine-tuning
```

BERT는 이 흐름의 대표입니다.

GPT-3는 여기서 한 단계 더 나아갑니다.

```text
pre-training + prompting
```

즉:

> 모델이 task마다 다시 학습되는 것이 아니라, prompt 안에서 task를 정의받고 수행하는 방향.

이것이 현대 LLM 사용 방식의 핵심입니다.

---

## 쉬운 비유: BERT와 GPT-3

### BERT

BERT는 “국어 독해를 잘하는 학생”에 가깝습니다.

```text
문장 전체를 읽고,
빈칸을 맞히고,
문장 의미를 이해하고,
질문에 해당하는 부분을 찾는다.
```

### GPT-3

GPT-3는 “글을 이어 쓰면서 과제를 수행하는 학생”에 가깝습니다.

```text
앞에 적힌 지시문과 예시를 보고,
다음에 올 말을 적절하게 이어 쓴다.
```

예를 들어 prompt가 번역 예시라면 번역을 이어 쓰고, 감성분석 예시라면 긍정/부정을 이어 쓰고, QA 예시라면 답변을 이어 씁니다.

즉, GPT-3의 기본 행동은 항상 이것입니다.

```text
다음에 올 텍스트를 예측한다.
```

하지만 prompt를 잘 구성하면, 이 다음 텍스트 예측이 다양한 task 수행처럼 보이게 됩니다.

---

## 입문자가 꼭 기억해야 할 문장

**GPT-3는 대규모 다음-token 예측 모델을 학습해두면, 새로운 task를 fine-tuning하지 않아도 prompt 안의 설명과 예시만으로 수행할 수 있다는 것을 보여준 논문입니다.**

더 짧게 말하면:

> **GPT-3 = next-token prediction으로 학습한 거대 Transformer가 prompt만 보고 여러 task를 수행할 수 있음을 보여준 모델**

**논문명:** Language Models are Few-Shot Learners  
**저자:** Tom B. Brown et al.  
**핵심 결론:** GPT-3는 175B parameter autoregressive language model이며, task-specific fine-tuning 없이 prompt 안의 instruction과 few-shot demonstration만으로 다양한 NLP task를 수행할 수 있음을 보여줬다.

**왜 중요함:** BERT 이후의 “pre-training + fine-tuning” 중심 패러다임에서, “pre-training + prompting / in-context learning”이라는 현대 LLM 사용 방식으로 넘어가는 결정적 논문이다.

**입문자가 배울 점:** GPT-3의 few-shot learning은 모델 weight를 업데이트하는 학습이 아니라, prompt 안에 제공된 예시를 보고 문맥 안에서 task 패턴을 파악하는 in-context learning이다.

**가장 중요한 문장:** GPT-3는 fine-tuning하지 않고도, 텍스트 prompt에 적힌 지시와 예시를 조건으로 삼아 다음 token을 생성함으로써 여러 task를 수행한다.
