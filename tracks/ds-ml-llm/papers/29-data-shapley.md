# Data Shapley: Equitable Valuation of Data for Machine Learning

## 결론부터

이 논문의 결론은 이겁니다.

**모델 성능에 각 학습 데이터가 얼마나 기여했는지를 공정하게 평가하려면, 각 데이터 포인트를 하나의 “기여자”로 보고 Shapley value 방식으로 가치를 나누면 된다.**  
이 방법이 바로 **Data Shapley**입니다.

한 줄로 말하면:

> **Data Shapley = 특정 학습 데이터 하나가 모델 성능 향상에 평균적으로 얼마나 기여했는지 계산하는 방법**

예를 들어 학습 데이터가 1,000개 있고, 이 데이터로 심장질환 예측 모델을 학습한다고 합시다. 이때 Data Shapley는 묻습니다.

```text
1번 환자 데이터는 모델 성능에 얼마나 기여했는가?
2번 환자 데이터는 모델 성능에 얼마나 기여했는가?
...
1000번 환자 데이터는 모델 성능에 얼마나 기여했는가?
```

---

## 이 논문을 한 문장으로 요약하면

**Data Shapley는 “이 데이터가 모델 성능에 얼마나 도움이 되었는가?”를 게임이론의 Shapley value로 계산하는 데이터 가치평가 방법입니다.**

지금까지 우리는 모델을 평가했습니다.

```text
accuracy
AUC
RMSE
F1-score
loss
```

이번 논문은 질문을 바꿉니다.

```text
모델이 얼마나 좋은가?
```

가 아니라:

```text
어떤 데이터가 이 모델을 좋게 만들었는가?
```

를 묻습니다.

---

## 왜 이 논문이 중요한가?

이전 논문들과 연결하면 위치가 분명합니다.

| 논문 | 핵심 질문 |
|---|---|
| **Datasheets for Datasets** | 데이터셋은 어떻게 만들어졌고, 어떤 한계가 있는가? |
| **Model Evaluation** | 모델 성능을 어떻게 공정하게 평가할 것인가? |
| **SHAP** | 특정 예측에서 feature가 얼마나 기여했는가? |
| **Data Shapley** | 특정 학습 데이터가 모델 성능에 얼마나 기여했는가? |

SHAP은 주로 **feature의 기여도**를 설명했습니다.

```text
이 고객의 이탈 예측에서 가입기간은 +0.18만큼 기여했다.
최근 사용량 감소는 +0.22만큼 기여했다.
```

Data Shapley는 **학습 데이터의 기여도**를 봅니다.

```text
이 학습 샘플은 모델 성능에 크게 기여했다.
이 학습 샘플은 오히려 모델 성능을 해쳤다.
이 학습 샘플은 중복이라 가치가 낮다.
```

즉, SHAP이 “예측 설명”에 가깝다면, Data Shapley는 “데이터 가치평가”에 가깝습니다.

---

## 핵심 개념 1: 데이터도 가치가 다르다

**모든 학습 데이터가 모델에 똑같이 중요한 것은 아닙니다.**

예를 들어 고양이와 강아지를 분류하는 이미지 데이터셋이 있다고 합시다.

어떤 데이터는 모델에 큰 도움이 됩니다.

```text
다양한 조명에서 찍힌 고양이 사진
애매한 품종의 강아지 사진
모델이 헷갈리기 쉬운 boundary 근처 샘플
```

어떤 데이터는 별 도움이 안 됩니다.

```text
이미 수천 장 있는 거의 똑같은 고양이 사진
완전히 중복된 이미지
너무 쉬운 샘플
```

어떤 데이터는 오히려 해롭습니다.

```text
고양이 사진인데 강아지라고 잘못 라벨링됨
이미지가 깨져 있음
다른 도메인에서 온 이상한 샘플
```

Data Shapley는 이런 차이를 정량화하려고 합니다.

```text
높은 Data Shapley value
→ 모델 성능에 많이 기여한 데이터

낮거나 음수인 Data Shapley value
→ 별로 도움이 안 되거나 모델 성능을 해친 데이터
```

---

## 핵심 개념 2: Shapley value

**Shapley value는 여러 참여자가 함께 만든 성과를 각 참여자의 평균 기여도에 따라 공정하게 나누는 방법입니다.**

이 개념은 게임이론에서 왔습니다.

예를 들어 세 사람이 팀 프로젝트를 했습니다.

```text
A, B, C가 함께 프로젝트를 해서 100점을 받았다.
```

문제는 이것입니다.

```text
A는 몇 점만큼 기여했나?
B는 몇 점만큼 기여했나?
C는 몇 점만큼 기여했나?
```

단순히 1/3씩 나누면 공정하지 않을 수 있습니다.

A가 혼자서도 많은 일을 했을 수 있고, B는 C와 함께 있을 때만 시너지가 났을 수 있습니다.

Shapley value는 각 사람이 여러 조합에 들어갔을 때 **추가로 얼마나 성과를 높였는지**를 평균내서 기여도를 계산합니다.

Data Shapley는 이 아이디어를 데이터에 적용합니다.

| 게임이론 Shapley | Data Shapley |
|---|---|
| 플레이어 | 학습 데이터 포인트 |
| 팀의 성과 | 모델 성능 |
| 플레이어 기여도 | 데이터가 모델 성능을 올린 정도 |
| Shapley value | Data Shapley value |

---

## 핵심 개념 3: Data Shapley는 무엇을 평균내는가?

**Data Shapley는 어떤 데이터가 여러 가능한 학습 데이터 subset에 추가될 때 성능을 얼마나 올리는지를 평균냅니다.**

학습 데이터셋이 있다고 합시다.

```text
D = {A, B, C, D}
```

우리가 A의 가치를 알고 싶습니다.

A를 단독으로 봐서는 안 됩니다.

왜냐하면 A의 가치는 다른 데이터와 함께 있을 때 달라질 수 있기 때문입니다.

예를 들어:

```text
A만 있을 때는 별로 도움 안 됨
B와 같이 있으면 도움 됨
C, D와 같이 있으면 중복이라 도움 적음
```

그래서 Data Shapley는 여러 상황을 봅니다.

```text
A를 빈 데이터셋에 추가하면 성능이 얼마나 오르나?
A를 {B}에 추가하면 성능이 얼마나 오르나?
A를 {C}에 추가하면 성능이 얼마나 오르나?
A를 {B, C}에 추가하면 성능이 얼마나 오르나?
...
```

이 모든 **marginal contribution**, 즉 추가 기여도를 평균낸 것이 A의 Data Shapley value입니다.

---

## 쉬운 예시: 세 개의 학습 데이터

학습 데이터가 세 개만 있다고 해봅시다.

```text
A, B, C
```

검증 성능을 `V(S)`라고 합시다.

```text
V({A}) = A만 가지고 학습한 모델 성능
V({A, B}) = A와 B로 학습한 모델 성능
```

A의 가치는 이렇게 봅니다.

```text
빈 상태에 A 추가: V({A}) - V({})
B만 있는 상태에 A 추가: V({A,B}) - V({B})
C만 있는 상태에 A 추가: V({A,C}) - V({C})
B,C가 있는 상태에 A 추가: V({A,B,C}) - V({B,C})
```

이 기여들을 적절히 평균낸 값이 A의 Data Shapley입니다.

즉, Data Shapley는 이렇게 묻습니다.

> A는 어떤 팀에 들어가든 평균적으로 성능을 얼마나 올리는가?

---

## 핵심 개념 4: Leave-One-Out이 왜 부족한가?

**Leave-One-Out은 데이터 하나를 전체 데이터셋에서 빼보고 성능이 얼마나 떨어지는지만 보는 방법인데, 데이터 간 상호작용과 중복을 잘 반영하지 못합니다.**

Leave-One-Out, 줄여서 LOO는 이렇게 합니다.

```text
전체 데이터로 모델 학습 → 성능 V(D)
데이터 i를 뺀 데이터로 모델 학습 → 성능 V(D - {i})
차이 = V(D) - V(D - {i})
```

이 차이가 크면 데이터 i가 중요하다고 봅니다.

겉보기에는 괜찮아 보입니다.

하지만 문제가 있습니다.

## 예시: 완전히 중복된 데이터

학습 데이터가 이렇게 있습니다.

```text
A, A', B, B', C, C'
```

여기서 `A`와 `A'`는 완전히 같은 데이터입니다.

A를 하나 제거해도 A'가 남아 있습니다.

```text
A 제거
→ A'가 여전히 있음
→ 모델 성능 변화 없음
```

그러면 Leave-One-Out은 A의 가치를 0으로 봅니다.

하지만 이상합니다.

A 계열 데이터가 모델에 아무 가치가 없는 것은 아닙니다.

A와 A'가 함께 중복되어 있어서 하나를 빼도 영향이 없을 뿐입니다.

---

## Data Shapley는 왜 더 낫나?

Data Shapley는 단순히 전체 데이터에서 하나를 빼보는 것이 아닙니다.

다양한 subset에 데이터가 추가될 때의 평균 기여를 봅니다.

그래서 중복 데이터가 있을 때도 가치를 더 공정하게 나눌 수 있습니다.

예를 들어 A와 A'가 같은 정보라면:

```text
A와 A'가 함께 정보 가치를 나눠 가짐
```

이라고 볼 수 있습니다.

즉, Data Shapley는 데이터 하나의 가치를 **전체 데이터셋 마지막 상태에서만** 평가하지 않고, **여러 가능한 데이터 조합 속에서의 평균적 기여**로 평가합니다.

---

## 핵심 개념 5: 공정한 데이터 가치평가의 세 가지 성질

논문은 Data Shapley가 만족해야 할 자연스러운 공정성 조건을 제시합니다.

크게 세 가지입니다.

```text
1. Null player
2. Symmetry
3. Additivity
```

## 1. Null player

**어떤 데이터가 어떤 상황에서도 성능을 전혀 바꾸지 않는다면, 그 데이터의 가치는 0이어야 합니다.**

## 2. Symmetry

**두 데이터가 모든 상황에서 똑같은 성능 변화를 만든다면, 두 데이터의 가치는 같아야 합니다.**

## 3. Additivity

**전체 성능 지표가 여러 개별 성능의 합이라면, 데이터의 전체 가치는 각 개별 성능에 대한 가치의 합이어야 합니다.**

---

## 핵심 개념 6: Data Shapley의 값은 맥락 의존적이다

**Data Shapley는 데이터 자체의 절대적 가치를 말하지 않습니다.**

어떤 데이터의 Shapley value는 다음에 따라 달라집니다.

```text
학습 알고리즘 A
평가 지표 V
함께 있는 다른 학습 데이터 D
평가용 test set 또는 target distribution
```

예를 들어 같은 데이터라도:

```text
logistic regression에서는 중요
neural network에서는 덜 중요
```

할 수 있습니다.

또는:

```text
accuracy를 볼 때는 중요
fairness metric을 볼 때는 덜 중요
```

할 수 있습니다.

즉, 데이터 가치는 항상 질문과 연결됩니다.

```text
무슨 모델을 학습하는가?
무슨 성능 지표로 평가하는가?
어떤 test population에 대해 평가하는가?
```

를 반드시 같이 봐야 합니다.

---

## 핵심 개념 7: 계산이 너무 비싸다

**정확한 Data Shapley를 계산하려면 가능한 모든 데이터 subset을 봐야 하므로, 실제 큰 데이터셋에서는 그대로 계산할 수 없습니다.**

데이터가 `n`개 있을 때 subset 수는:

```text
2^n
```

입니다.

데이터가 10개면 subset이 1,024개입니다.

데이터가 100개면 사실상 불가능합니다.

그리고 각 subset마다 모델을 다시 학습해야 합니다.

```text
subset S로 모델 학습
성능 V(S) 계산
```

이건 너무 비쌉니다.

그래서 논문은 근사 알고리즘을 제안합니다.

```text
1. Truncated Monte Carlo Shapley, TMC-Shapley
2. Gradient-based Shapley, G-Shapley
```

---

## 핵심 개념 8: TMC-Shapley

**TMC-Shapley는 데이터 순서를 랜덤하게 여러 번 섞고, 그 순서대로 데이터를 하나씩 추가하면서 성능 증가분을 평균내는 방법입니다.**

예를 들어 데이터가 있습니다.

```text
A, B, C, D
```

랜덤 순서를 하나 뽑습니다.

```text
C → A → D → B
```

처음에는 빈 데이터셋입니다.

```text
S = {}
```

C를 추가합니다.

```text
V({C}) - V({})
```

A를 추가합니다.

```text
V({C, A}) - V({C})
```

D를 추가합니다.

```text
V({C, A, D}) - V({C, A})
```

B를 추가합니다.

```text
V({C, A, D, B}) - V({C, A, D})
```

이 기여도를 기록합니다.

그다음 다른 랜덤 순서를 뽑습니다.

```text
B → D → A → C
```

또 같은 과정을 반복합니다.

여러 랜덤 순서에서 평균을 내면 Shapley value를 근사할 수 있습니다.

---

## TMC-Shapley에서 truncation은 왜 필요한가?

데이터를 하나씩 추가하다 보면 어느 순간 모델 성능이 전체 데이터 성능에 거의 가까워집니다.

```text
V(S) ≈ V(D)
```

이후 데이터를 더 추가해도 성능 변화가 매우 작을 수 있습니다.

그러면 나머지 계산을 생략할 수 있습니다.

```text
이미 전체 성능에 거의 도달
→ 나머지 marginal contribution을 0에 가깝게 처리
→ 계산 절약
```

---

## 핵심 개념 9: G-Shapley

**G-Shapley는 gradient 정보를 이용해 Data Shapley를 더 빠르게 근사하려는 방법입니다.**

TMC-Shapley는 여전히 많은 모델 재학습이 필요합니다.

큰 neural network에서는 이것도 비쌉니다.

그래서 논문은 gradient 기반 근사도 제안합니다.

직관은 이렇습니다.

```text
데이터 하나가 학습 과정에서 gradient update에 얼마나 기여했는가?
그 기여가 성능 변화와 어떻게 연결되는가?
```

즉, 모델을 처음부터 매 subset마다 다시 학습하지 않고, 학습 과정의 gradient 정보를 이용해 데이터의 가치를 근사합니다.

입문 단계에서는 이렇게만 이해하면 충분합니다.

```text
TMC-Shapley = 여러 subset을 실제로 학습해가며 근사
G-Shapley = gradient 정보를 이용해 더 빠르게 근사
```

---

## Data Shapley로 무엇을 할 수 있나?

이 논문이 중요한 이유는 단순히 “가치 점수”를 계산하는 데서 끝나지 않기 때문입니다.

논문은 Data Shapley가 여러 실용적 용도를 가진다고 보여줍니다.

```text
1. 중요한 데이터 찾기
2. mislabeled data / corrupted data 찾기
3. 어떤 데이터를 더 모을지 결정하기
4. 그룹별 데이터 가치 분석하기
```

---

## 활용 1: 중요한 학습 데이터 찾기

높은 Data Shapley value를 가진 데이터는 모델 성능에 중요합니다.

예를 들어 이미지 분류에서 높은 가치 데이터는 다음일 수 있습니다.

```text
모델이 헷갈리기 쉬운 boundary 근처 샘플
희귀하지만 중요한 패턴
다양성을 높이는 샘플
평가 데이터와 잘 맞는 샘플
```

---

## 활용 2: 잘못 라벨링된 데이터 찾기

낮거나 음수인 Shapley value는 특히 중요합니다.

음수라는 것은 대략 이런 의미입니다.

```text
이 데이터는 평균적으로 모델 성능을 낮춘다.
```

그 이유는 다음일 수 있습니다.

```text
라벨이 틀림
데이터가 outlier
입력이 깨짐
다른 distribution에서 옴
학습에 혼란을 줌
```

---

## 활용 3: 어떤 데이터를 더 수집할지 결정하기

데이터 수집에는 비용이 듭니다.

예를 들어 의료 AI 모델을 만든다고 합시다.

더 많은 데이터를 모으고 싶지만, 비용이 큽니다.

```text
젊은 남성 환자 데이터
고령 여성 환자 데이터
특정 인종 그룹 데이터
희귀 질환 환자 데이터
```

어떤 데이터를 더 모아야 모델 성능이 좋아질까요?

Data Shapley는 기존 데이터에서 높은 가치 그룹을 찾아서, 어떤 유형의 데이터가 모델 성능에 중요한지 힌트를 줍니다.

---

## 활용 4: 그룹 단위 Data Shapley

**개별 데이터 하나가 아니라, 데이터 그룹의 가치를 계산할 수도 있습니다.**

실무에서는 개별 샘플보다 그룹의 가치가 더 중요할 때가 많습니다.

예를 들어 의료 데이터에서:

```text
성별 × 인종 × 나이 그룹
```

별로 데이터를 나눌 수 있습니다.

그룹 예:

```text
고령 여성 환자
젊은 남성 환자
특정 인종의 중년 환자
```

각 그룹이 모델 성능에 얼마나 기여했는지 볼 수 있습니다.

이건 공정성 분석과도 연결됩니다.

```text
어떤 집단의 데이터가 모델 성능에 크게 기여하는가?
어떤 집단의 데이터가 부족한가?
어떤 집단을 더 수집해야 하는가?
```

---

## Data Shapley와 SHAP의 차이

| 구분 | SHAP | Data Shapley |
|---|---|---|
| 평가 대상 | feature | training data point |
| 설명 대상 | 특정 예측값 | 모델 성능에 대한 데이터 기여 |
| 질문 | 이 예측에서 어떤 feature가 영향을 줬나? | 이 학습 데이터가 모델 성능에 얼마나 기여했나? |
| 기반 개념 | Shapley value | Shapley value |
| 활용 | 모델 해석, feature attribution | 데이터 가치평가, 데이터 클리닝, 데이터 수집 전략 |

예를 들어 고객 이탈 모델이 있습니다.

## SHAP 질문

```text
이 고객의 이탈 예측에서 어떤 feature가 중요했나?
```

답:

```text
최근 사용량 감소가 +0.20
불만 접수가 +0.15
장기 고객 여부가 -0.10
```

## Data Shapley 질문

```text
이 모델을 학습하는 데 어떤 학습 데이터가 중요했나?
```

답:

```text
고객 데이터 123번은 높은 가치
고객 데이터 456번은 라벨 오류 가능성
고객 데이터 789번은 중복이라 낮은 가치
```

둘 다 Shapley value를 쓰지만, 적용 대상이 다릅니다.

---

## Data Shapley와 Leave-One-Out 비교

| 구분 | Leave-One-Out | Data Shapley |
|---|---|---|
| 기본 방식 | 전체 데이터에서 하나 빼기 | 모든 subset에서 평균 marginal contribution 계산 |
| 계산 | 상대적으로 단순 | 정확 계산은 지수적으로 비쌈 |
| 데이터 상호작용 | 잘 반영 못 함 | 잘 반영하려고 함 |
| 중복 데이터 처리 | 가치가 0으로 나올 수 있음 | 중복 데이터끼리 가치 나눔 |
| 공정성 성질 | Shapley 성질 불만족 가능 | null, symmetry, additivity 만족 |
| 실무 사용 | 빠른 baseline | 더 원칙적이지만 근사 필요 |

---

## Data Shapley의 한계

## 1. 계산 비용이 크다

정확한 Shapley value는 모든 subset을 봐야 합니다.

```text
2^n subset
각 subset마다 모델 학습
```

현실적으로 불가능합니다.

TMC-Shapley나 G-Shapley 같은 근사가 필요합니다.

## 2. 데이터 가치가 절대적이지 않다

Data Shapley value는 다음에 의존합니다.

```text
학습 알고리즘
평가 지표
test set
같이 있는 다른 데이터
```

그래서 어떤 데이터가 한 프로젝트에서 낮은 가치라고 해서, 본질적으로 쓸모없는 데이터라는 뜻은 아닙니다.

## 3. 개인정보의 본질적 가치는 반영하지 않는다

Data Shapley는 모델 성능 기여도를 측정합니다.

하지만 개인 데이터의 가치는 그것만이 아닙니다.

```text
프라이버시
민감성
개인에게 주는 위험
법적 권리
사회적 맥락
동의 여부
```

이런 것은 Data Shapley value에 자동으로 들어가지 않습니다.

## 4. 평가 지표가 잘못되면 가치도 잘못된다

Data Shapley는 성능 지표 `V`를 기준으로 계산합니다.

그런데 `V`가 잘못된 지표라면, Shapley value도 잘못된 방향을 가질 수 있습니다.

---

## 실무에서 Data Shapley를 어떻게 쓰면 좋나?

## 1. 데이터 클리닝 우선순위 정하기

데이터가 너무 많아서 전부 검수할 수 없을 때:

```text
낮은 Shapley value 데이터부터 검수
```

하면 라벨 오류나 이상치를 더 빨리 찾을 수 있습니다.

## 2. 데이터 구매 또는 수집 전략 세우기

어떤 종류의 데이터가 모델 성능에 중요한지 보고:

```text
비슷한 high-value 데이터 더 수집
부족한 그룹 데이터 보강
```

에 활용할 수 있습니다.

## 3. 데이터셋 품질 분석

데이터셋에서:

```text
고가치 데이터
중복 데이터
해로운 데이터
```

를 구분할 수 있습니다.

## 4. 그룹별 데이터 기여 분석

공정성이나 도메인 coverage 관점에서:

```text
어떤 인구집단 데이터가 모델에 중요했는가?
어떤 집단 데이터가 부족한가?
```

를 볼 수 있습니다.

---

## 쉬운 실무 예시: 이미지 분류 데이터셋 정리

강아지 vs 고양이 이미지 분류 모델을 만든다고 합시다.

학습 데이터가 10만 장 있습니다.

모델 성능이 생각보다 낮습니다.

Data Shapley를 계산하거나 근사합니다.

결과:

```text
높은 value:
- 다양한 각도의 고양이 이미지
- 어두운 조명에서 찍힌 강아지 이미지
- 헷갈리기 쉬운 품종 이미지

낮은 value:
- 거의 같은 이미지 중복
- 라벨이 틀린 이미지
- 너무 흐릿하거나 깨진 이미지
```

이제 할 수 있는 일:

```text
낮은 value 데이터 검수
라벨 오류 수정
중복 제거
high-value 유형의 데이터 추가 수집
```

이렇게 하면 단순히 모델을 더 크게 만드는 것이 아니라, 데이터셋 자체를 개선할 수 있습니다.

이것이 데이터 중심 ML의 관점입니다.

---

## 이 논문이 데이터 중심 ML에서 중요한 이유

지금까지 모델 중심으로 보면 이렇게 생각하기 쉽습니다.

```text
성능이 낮다
→ 더 좋은 모델을 쓰자
→ XGBoost, LightGBM, Transformer, 더 큰 LLM
```

Data Shapley는 다른 질문을 던집니다.

```text
성능이 낮다
→ 어떤 데이터가 성능을 올리고 있나?
→ 어떤 데이터가 성능을 해치고 있나?
→ 어떤 데이터를 더 모아야 하나?
```

즉, 모델을 바꾸기 전에 데이터의 가치를 분석합니다.

이것이 데이터 중심 ML의 핵심입니다.

## 이전 논문들과 연결해서 이해하기

| 이전 논문 | Data Shapley와의 연결 |
|---|---|
| **Tidy Data** | 데이터 구조가 좋아야 개별 데이터의 가치도 의미 있게 분석 가능 |
| **Model Evaluation** | Data Shapley는 특정 평가 지표 `V`에 대한 데이터 기여도 |
| **Datasheets** | 데이터셋의 출처와 한계를 문서화하고, Data Shapley로 성능 기여도 분석 가능 |
| **Model Cards** | 모델 성능과 한계 분석에 데이터 가치 정보를 보조로 활용 가능 |
| **SHAP** | SHAP은 feature attribution, Data Shapley는 data attribution |
| **Hidden Technical Debt** | low-value, unstable, corrupted data를 찾아 data debt를 줄일 수 있음 |
| **YouTube Recommendations** | 추천 시스템에서 어떤 사용자 행동 데이터가 성능에 기여하는지 분석 가능 |
| **Collaborative Filtering** | user-item interaction 중 어떤 interaction이 추천 성능에 가치 있는지 평가 가능 |

## 이 논문이 주는 진짜 메시지

이 논문의 진짜 메시지는 단순히 “Shapley value를 데이터에 적용했다”가 아닙니다.

더 중요한 메시지는 이것입니다.

**데이터는 단순히 많을수록 좋은 것이 아니다.**  
**각 데이터는 모델 성능에 서로 다른 기여를 하며, 일부 데이터는 중복이고, 일부 데이터는 핵심적이며, 일부 데이터는 오히려 해로울 수 있다.**

따라서 좋은 데이터사이언티스트는 이렇게 물어야 합니다.

```text
어떤 데이터를 더 모아야 하는가?
어떤 데이터를 제거해야 하는가?
어떤 데이터가 mislabeled일 가능성이 높은가?
어떤 데이터가 특정 metric에 가장 중요한가?
데이터의 가치는 어떤 algorithm과 metric 아래에서 정의되는가?
```

이 질문들이 데이터 중심 ML로 가는 핵심입니다.

---

## 입문자가 꼭 기억해야 할 문장

**Data Shapley는 각 학습 데이터가 여러 가능한 데이터 subset에 추가될 때 모델 성능을 얼마나 올리는지 평균내어, 그 데이터의 공정한 기여도를 계산하는 방법입니다.**

더 짧게 말하면:

> **Data Shapley = 학습 데이터 하나하나의 모델 성능 기여도**

## 오늘 공부용 요약

**논문명:** Data Shapley: Equitable Valuation of Data for Machine Learning  
**저자:** Amirata Ghorbani, James Zou  
**출판:** ICML 2019

**핵심 결론:** Data Shapley는 학습 데이터 하나하나의 가치를 Shapley value로 계산하는 방법입니다. 각 데이터가 여러 가능한 subset에 추가될 때 성능을 얼마나 올리는지 평균내어, 모델 성능에 대한 공정한 기여도를 평가합니다.

**왜 중요함:** 모델 성능을 높이기 위해 항상 모델만 바꿀 필요는 없습니다. Data Shapley는 어떤 데이터가 중요하고, 어떤 데이터가 중복이며, 어떤 데이터가 라벨 오류나 outlier일 가능성이 높은지 알려줄 수 있습니다.

**입문자가 배울 점:** 데이터의 가치는 절대적이지 않습니다. 특정 학습 알고리즘, 평가 지표, 학습 데이터 구성, test distribution에 따라 달라집니다. 따라서 Data Shapley를 해석할 때는 항상 “어떤 모델과 어떤 metric에 대한 가치인가?”를 같이 봐야 합니다.

**가장 중요한 문장:** Data Shapley는 “데이터가 많다”가 아니라 “어떤 데이터가 모델 성능에 실제로 기여하는가”를 묻는 데이터 중심 ML의 핵심 도구입니다.
