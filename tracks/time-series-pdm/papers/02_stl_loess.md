# 2편: STL: A Seasonal-Trend Decomposition Procedure Based on Loess

## 결론부터

**이 논문의 핵심 결론은, 시계열을 원본 그대로 보지 말고 “추세 + 계절성 + 잔차”로 분해하면 정상적인 반복 패턴과 진짜 이상 신호를 더 명확히 구분할 수 있다는 것입니다.**

## 이 논문을 한 문장으로 요약하면

**STL은 LOESS라는 부드러운 지역 회귀 기법을 반복적으로 사용해 시계열을 trend, seasonal, remainder로 나누는 유연한 분해 방법입니다.** 논문은 STL의 목표를 단순한 사용법, 계절성과 추세의 유연한 조절, 임의의 계절 주기, 결측값 처리, 이상치에 덜 흔들리는 robust decomposition, 긴 시계열에서도 빠른 계산으로 정리합니다.

참고 링크: https://www.math.unm.edu/~lil/Stat581/STL.pdf

---

## 왜 이 논문이 필요했나?

1편 ARIMA에서는 시계열을 예측하기 위해 **정상성, 차분, 자기상관, 잔차**를 봤습니다. 그런데 실무 센서 데이터에서는 차분만으로 충분하지 않은 경우가 많습니다.

예를 들어 공장 온도 센서를 생각해보겠습니다.

| 시간 | 온도 |
|---|---:|
| 00:00 | 62 |
| 06:00 | 61 |
| 12:00 | 68 |
| 18:00 | 70 |
| 24:00 | 63 |

이 온도 변화가 이상일까요? 아닐 수 있습니다. 낮에는 외기 온도와 설비 부하가 올라가고, 밤에는 내려가는 **정상적인 하루 주기**일 수 있습니다.

이번에는 압축기 압력 데이터를 보겠습니다.

| 시간 | 압력 |
|---|---:|
| 09:00 | 5.0 |
| 09:01 | 5.4 |
| 09:02 | 5.8 |
| 09:03 | 5.3 |
| 09:04 | 5.0 |
| 09:05 | 5.5 |
| 09:06 | 5.9 |
| 09:07 | 5.4 |

이것도 이상이 아닐 수 있습니다. 압축기가 켜지고 꺼지는 주기 때문에 반복적으로 오르내릴 수 있습니다.

문제는 여기서 갑자기 값이 튀는 경우입니다.

| 시간 | 압력 |
|---|---:|
| 09:00 | 5.0 |
| 09:01 | 5.4 |
| 09:02 | 5.8 |
| 09:03 | 5.3 |
| 09:04 | 5.0 |
| 09:05 | 5.5 |
| 09:06 | 7.9 |
| 09:07 | 5.4 |

09:06의 7.9는 진짜 이상일 수 있습니다. 하지만 원본만 보면 “주기적으로 오르내리는 정상 패턴”과 “진짜 튄 값”이 섞여 있습니다.

STL이 필요한 이유가 바로 여기에 있습니다.

**원본 시계열 = 장기 추세 + 반복 패턴 + 설명되지 않는 잔차**

이렇게 나누면, 설비 데이터에서 다음 질문에 답하기 쉬워집니다.

| 질문 | STL로 보는 방법 |
|---|---|
| 설비가 서서히 열화되고 있나? | trend 확인 |
| 정상적인 운전 주기가 있나? | seasonal 확인 |
| 갑작스러운 이상이 있었나? | remainder 확인 |
| 예측 모델이 놓친 비정상 패턴이 있나? | residual/remainder 확인 |
| 경보 기준을 어디에 둘까? | remainder의 분포 확인 |

## 핵심 개념 1. 시계열 분해: 원본을 세 조각으로 나눈다

STL의 기본 생각은 매우 단순합니다.

\[
Y_t = T_t + S_t + R_t
\]

| 기호 | 의미 | 설비 예시 |
|---|---|---|
| \(Y_t\) | 원본 센서값 | 실제 온도, 압력, 전류, 진동 RMS |
| \(T_t\) | trend | 설비가 장기적으로 뜨거워지는 추세 |
| \(S_t\) | seasonal | 매일 반복되는 운전 패턴, 교대조 주기, 압축기 on/off 주기 |
| \(R_t\) | remainder | 추세와 주기로 설명되지 않는 잔차 |

쉽게 비유하면 이렇습니다.

센서 원본값은 **완성된 음악**입니다.  
STL은 이 음악을 다음처럼 나눕니다.

| 음악 비유 | STL 구성요소 |
|---|---|
| 곡 전체의 분위기가 점점 고조됨 | trend |
| 반복되는 드럼 리듬 | seasonal |
| 순간적인 잡음, 삑사리, 예상 밖 소리 | remainder |

설비 진단에서 우리가 특히 관심 갖는 것은 대개 remainder입니다. 정상적인 추세와 정상적인 반복 패턴을 제거했는데도 크게 남는 값은 이상 후보가 될 수 있기 때문입니다.

---

## 핵심 개념 2. Trend는 “서서히 변하는 큰 흐름”이다

trend는 장기적인 방향입니다.

예를 들어 모터 베어링 온도가 아래처럼 변한다고 하겠습니다.

| 날짜 | 평균 온도 |
|---|---:|
| 월 | 61 |
| 화 | 62 |
| 수 | 63 |
| 목 | 65 |
| 금 | 66 |

하루 안에서는 온도가 오르내릴 수 있지만, 전체적으로는 올라가고 있습니다. 이게 trend입니다.

설비 관점에서 trend는 매우 중요합니다.

| trend 형태 | 실무 해석 |
|---|---|
| 서서히 증가하는 온도 | 냉각 성능 저하, 마찰 증가, 윤활 문제 가능성 |
| 서서히 증가하는 진동 RMS | 베어링 마모, 정렬 불량, 불평형 증가 가능성 |
| 서서히 감소하는 유량 | 필터 막힘, 펌프 성능 저하 가능성 |
| 서서히 증가하는 전류 | 부하 증가, 기계적 저항 증가 가능성 |

단, trend가 있다고 해서 무조건 고장은 아닙니다. 생산량이 늘거나 외기 온도가 높아졌거나 운전 모드가 바뀌어도 trend가 변합니다. 그래서 trend를 해석할 때는 반드시 **운전 조건, 부하, 회전수, 제품 종류, 외기 온도**를 같이 봐야 합니다.

---

## 핵심 개념 3. Seasonal은 “정상적으로 반복되는 패턴”이다

seasonal이라는 단어 때문에 계절, 즉 봄·여름·가을·겨울만 떠올리기 쉽습니다. 하지만 시계열에서 seasonal은 더 넓은 의미입니다.

**일정한 주기로 반복되는 패턴**입니다.

설비 데이터에서는 이런 것들이 모두 seasonal이 될 수 있습니다.

| 반복 주기 | 설비 예시 |
|---|---|
| 1초 | 회전체 1회전 주기 |
| 1분 | 압축기 제어 주기 |
| 10분 | batch 공정 반복 |
| 1시간 | 설비 예열·냉각 사이클 |
| 8시간 | 교대조별 운전 패턴 |
| 24시간 | 주야간 온도·부하 차이 |
| 7일 | 평일/주말 생산량 차이 |

STL 논문에서는 월별 데이터라면 1월끼리, 2월끼리, 3월끼리 따로 모아 seasonal subseries를 만든다는 방식으로 설명합니다.

설비 데이터로 바꿔 생각하면, 24시간 주기가 있는 온도 센서에서는 매일 00:00 값끼리, 01:00 값끼리, 02:00 값끼리 비교하는 느낌입니다. 이렇게 하면 “매일 오후 2시에 온도가 높은 것은 정상 패턴인지”를 따로 볼 수 있습니다.

---

## 핵심 개념 4. Remainder는 “설명되지 않고 남은 것”이다

STL에서 remainder는 다음처럼 계산됩니다.

\[
R_t = Y_t - T_t - S_t
\]

직관은 간단합니다.

1. 원본값에서 장기 추세를 뺍니다.
2. 반복되는 정상 패턴도 뺍니다.
3. 그래도 남는 것이 remainder입니다.

예를 들어 이런 데이터가 있다고 하겠습니다.

| 시간 | 원본 온도 \(Y_t\) | 추세 \(T_t\) | 반복 패턴 \(S_t\) | 잔차 \(R_t\) |
|---|---:|---:|---:|---:|
| 09:00 | 70.0 | 70.0 | 0.0 | 0.0 |
| 09:01 | 70.5 | 70.1 | 0.4 | 0.0 |
| 09:02 | 70.9 | 70.2 | 0.7 | 0.0 |
| 09:03 | 70.4 | 70.3 | 0.1 | 0.0 |
| 09:04 | 75.5 | 70.4 | 0.4 | 4.7 |

09:04의 원본 온도 75.5만 보면 높아 보입니다. 하지만 더 중요한 것은 trend와 seasonal을 제거한 뒤에도 4.7이 남았다는 점입니다. 이것은 정상 반복 패턴으로 설명되지 않는 이상 후보입니다.

설비 이상탐지에서 STL을 쓰는 가장 쉬운 방법은 다음입니다.

1. 정상 운전 구간으로 STL을 적용한다.
2. remainder의 평균과 분산 또는 분위수를 본다.
3. 새 데이터의 remainder가 평소 범위를 벗어나면 경보 후보로 본다.

하지만 주의해야 합니다. remainder가 크다고 항상 고장은 아닙니다. 센서 순간 오류, 통신 누락, 운전 조건 변경, setpoint 변경, 정비 직후 재가동도 remainder를 키울 수 있습니다.

## 핵심 개념 5. LOESS: 가까운 점들을 더 믿고 부드러운 선을 그린다

STL의 핵심 도구는 LOESS입니다. LOESS를 쉽게 말하면 이렇습니다.

**전체 데이터에 하나의 직선을 억지로 그리지 말고, 각 시점 주변의 가까운 데이터들을 더 많이 반영해서 부드러운 선을 그리는 방법**입니다.

예를 들어 다음 온도 데이터를 보겠습니다.

| 시간 | 온도 |
|---|---:|
| 09:00 | 70.0 |
| 09:01 | 70.1 |
| 09:02 | 70.4 |
| 09:03 | 70.6 |
| 09:04 | 72.5 |
| 09:05 | 70.9 |
| 09:06 | 71.0 |

09:04에 72.5라는 튄 값이 있습니다. 단순 이동평균은 이 튄 값에 쉽게 끌려갈 수 있습니다. LOESS는 주변 점들을 사용해 부드러운 흐름을 추정하되, 거리와 가중치를 조절할 수 있습니다.

논문에서 LOESS는 특정 시점 \(x\) 주변의 가까운 \(q\)개 점을 선택하고, 가까운 점에 큰 가중치, 먼 점에 작은 가중치를 줍니다. 이때 tricube weight function을 사용합니다.

\[
W(u) =
\begin{cases}
(1-u^3)^3, & 0 \le u < 1 \\
0, & u \ge 1
\end{cases}
\]

수식이 조금 낯설지만 의미는 단순합니다.

| \(u\) | 의미 | 가중치 |
|---:|---|---|
| 0에 가까움 | 현재 시점과 매우 가까움 | 크게 반영 |
| 1에 가까움 | 현재 시점과 멀어짐 | 작게 반영 |
| 1 이상 | 너무 멂 | 반영하지 않음 |

설비 데이터로 보면, 10:00의 정상 trend를 추정할 때 10:01, 09:59 같은 가까운 시점은 많이 참고하고, 너무 멀리 떨어진 시점은 덜 참고하는 방식입니다.

## 핵심 개념 6. Cycle-subseries: 같은 위치끼리 모아서 계절성을 본다

STL의 재미있는 아이디어는 seasonal pattern을 볼 때 **같은 주기 위치끼리 따로 모아본다**는 것입니다.

월별 매출 데이터라면 이렇게 봅니다.

| cycle position | 모아보는 값 |
|---|---|
| 1월 | 2020년 1월, 2021년 1월, 2022년 1월 |
| 2월 | 2020년 2월, 2021년 2월, 2022년 2월 |
| 3월 | 2020년 3월, 2021년 3월, 2022년 3월 |

설비 온도 데이터가 24시간 주기를 가진다면 이렇게 볼 수 있습니다.

| cycle position | 모아보는 값 |
|---|---|
| 매일 00:00 | 월 00:00, 화 00:00, 수 00:00 |
| 매일 01:00 | 월 01:00, 화 01:00, 수 01:00 |
| 매일 02:00 | 월 02:00, 화 02:00, 수 02:00 |

압축기 압력 데이터가 4분 주기를 가진다면 이렇게 볼 수 있습니다.

| cycle position | 모아보는 값 |
|---|---|
| 주기 내 1번째 분 | 09:00, 09:04, 09:08 |
| 주기 내 2번째 분 | 09:01, 09:05, 09:09 |
| 주기 내 3번째 분 | 09:02, 09:06, 09:10 |
| 주기 내 4번째 분 | 09:03, 09:07, 09:11 |

이 방식의 장점은 seasonal pattern이 고정된 값이 아니라 **시간에 따라 조금씩 변할 수 있다**는 점을 허용한다는 것입니다. 예를 들어 매일 오후 2시에 온도가 높다는 패턴은 유지되지만, 여름이 되면 그 높이가 조금 더 커질 수 있습니다. STL은 이런 변화를 부드럽게 따라갈 수 있습니다.

## 핵심 개념 7. Inner loop: seasonal과 trend를 번갈아 업데이트한다

STL은 한 번 계산하고 끝나는 단순한 필터가 아닙니다. STL은 inner loop와 outer loop라는 두 개의 반복 절차로 구성됩니다. inner loop에서는 seasonal component와 trend component가 업데이트되고, outer loop에서는 이상치의 영향을 줄이기 위한 robust weight가 계산됩니다.

inner loop를 쉬운 말로 표현하면 이렇습니다.

1. 일단 trend를 빼본다.
2. 남은 데이터에서 seasonal pattern을 추정한다.
3. seasonal pattern을 빼본다.
4. 남은 데이터에서 trend를 다시 추정한다.
5. 이 과정을 반복한다.

왜 번갈아 해야 할까요?

trend와 seasonal은 서로 헷갈릴 수 있기 때문입니다.

예를 들어 온도가 여름에 높아지는 것은 seasonal일까요, trend일까요?  
설비가 마모되어 서서히 뜨거워지는 것은 trend일까요, seasonal일까요?

한 번에 완벽히 나누기 어렵습니다. 그래서 STL은 “seasonal을 추정하고, trend를 추정하고, 다시 seasonal을 고치고, 다시 trend를 고치는” 반복 구조를 사용합니다.

---

## 핵심 개념 8. Outer loop: 이상치가 분해 결과를 망치지 않게 한다

현장 센서 데이터에는 이상치가 많습니다.

| 원인 | 예시 |
|---|---|
| 센서 오류 | 순간적으로 999.9 기록 |
| 통신 오류 | 값이 0으로 떨어짐 |
| 재가동 | 정지 후 급격한 변화 |
| 외란 | 문 개방, 부하 급변 |
| 실제 고장 | 베어링 충격, 압력 spike |

문제는 이상치 하나가 trend나 seasonal 추정 자체를 망칠 수 있다는 것입니다. 예를 들어 하루 중 특정 시간에 센서값이 한 번 크게 튀었는데, 모델이 이것을 “매일 반복되는 정상 seasonal pattern”이라고 착각하면 안 됩니다.

그래서 STL은 outer loop에서 robustness weight를 계산합니다. remainder가 너무 큰 시점은 가중치를 낮춰 다음 smoothing에서 덜 반영되게 합니다.

간단히 말하면 다음과 같습니다.

| 시점 | remainder 크기 | 다음 반복에서의 영향 |
|---|---:|---|
| 평범한 값 | 작음 | 정상적으로 반영 |
| 조금 튄 값 | 중간 | 일부만 반영 |
| 극단적으로 튄 값 | 큼 | 거의 반영하지 않음 |

이 부분은 설비 이상탐지에서 매우 중요합니다. STL은 이상치를 완전히 해결하는 모델은 아니지만, 적어도 튄 값 하나 때문에 trend와 seasonal component 전체가 왜곡되는 것을 줄이려는 구조를 갖고 있습니다.

---

## 핵심 개념 9. STL은 예측 모델이라기보다 “분해 도구”다

ARIMA는 기본적으로 예측 모델입니다.  
STL은 기본적으로 분해 도구입니다.

즉, STL 자체가 “내일 값은 얼마다”를 직접 예측하는 모델이라기보다는, 원본 데이터를 더 이해하기 쉬운 세 조각으로 나눠주는 도구입니다.

하지만 예측과 결합할 수 있습니다.

예를 들어:

1. STL로 원본을 trend, seasonal, remainder로 나눈다.
2. trend는 단순 선형모델이나 ARIMA로 예측한다.
3. seasonal은 최근 반복 패턴을 유지한다고 가정한다.
4. remainder는 평균 0에 가까운 잡음으로 보거나 별도 모델로 예측한다.
5. 다시 합쳐서 최종 예측값을 만든다.

실무에서는 다음처럼 씁니다.

| 목적 | STL 사용법 |
|---|---|
| EDA | 센서값을 trend/seasonal/remainder로 시각화 |
| 이상탐지 | remainder가 큰 시점 탐지 |
| 특징 추출 | trend slope, seasonal amplitude, residual std 사용 |
| 전처리 | 계절성 제거 후 ML 모델 학습 |
| 예측 | STL + ARIMA, STL + XGBoost, STL + LSTM 등으로 결합 |

Python에서는 `statsmodels`의 `STL`이 trend, seasonal, residual 세 component로 분해하는 기능을 제공합니다.

## 작은 예시 데이터: 압축기 압력 센서 1분 단위 데이터

압축기 압력 센서가 4분 주기로 오르내린다고 하겠습니다. 그리고 설비가 조금씩 열화되어 전체 압력 수준이 서서히 올라간다고 가정하겠습니다.

| 시간 | 원본 압력 |
|---|---:|
| 09:00 | 5.0 |
| 09:01 | 5.5 |
| 09:02 | 6.0 |
| 09:03 | 5.4 |
| 09:04 | 5.1 |
| 09:05 | 5.6 |
| 09:06 | 6.1 |
| 09:07 | 5.5 |
| 09:08 | 5.2 |
| 09:09 | 5.7 |
| 09:10 | 7.8 |
| 09:11 | 5.6 |

여기에는 세 가지가 섞여 있습니다.

| 성분 | 설명 |
|---|---|
| trend | 시간이 지나며 기본 압력이 5.0 → 5.2 정도로 조금 상승 |
| seasonal | 4분마다 5.0 → 5.5 → 6.0 → 5.4 패턴 반복 |
| anomaly/remainder | 09:10의 7.8이 비정상적으로 큼 |

STL을 적용하면 대략 이런 식으로 해석할 수 있습니다.

| 시간 | 원본 압력 | trend | seasonal | remainder |
|---|---:|---:|---:|---:|
| 09:00 | 5.0 | 5.0 | 0.0 | 0.0 |
| 09:01 | 5.5 | 5.0 | 0.5 | 0.0 |
| 09:02 | 6.0 | 5.0 | 1.0 | 0.0 |
| 09:03 | 5.4 | 5.0 | 0.4 | 0.0 |
| 09:04 | 5.1 | 5.1 | 0.0 | 0.0 |
| 09:05 | 5.6 | 5.1 | 0.5 | 0.0 |
| 09:06 | 6.1 | 5.1 | 1.0 | 0.0 |
| 09:07 | 5.5 | 5.1 | 0.4 | 0.0 |
| 09:08 | 5.2 | 5.2 | 0.0 | 0.0 |
| 09:09 | 5.7 | 5.2 | 0.5 | 0.0 |
| 09:10 | 7.8 | 5.2 | 1.0 | 1.6 |
| 09:11 | 5.6 | 5.2 | 0.4 | 0.0 |

09:10의 원본 압력 7.8은 “주기상 원래 높아질 타이밍”이긴 합니다. 하지만 trend 5.2와 seasonal 1.0을 감안해도 1.6이 남습니다. 그래서 이 시점은 이상 후보입니다.

중요한 것은 이것입니다.

**원본값만 보면 09:02의 6.0도 높고, 09:10의 7.8도 높습니다. 하지만 STL로 보면 09:02는 정상 주기이고, 09:10은 잔차가 큰 이상 후보입니다.**

## 수식으로 보는 STL의 핵심

수식은 마지막에 정리하겠습니다. 먼저 직관부터 다시 잡으면 다음과 같습니다.

STL은 원본을 이렇게 나눕니다.

\[
Y_t = T_t + S_t + R_t
\]

잔차는 다음처럼 계산합니다.

\[
R_t = Y_t - T_t - S_t
\]

LOESS에서는 가까운 점에 더 큰 가중치를 줍니다.

\[
W(u) =
\begin{cases}
(1-u^3)^3, & 0 \le u < 1 \\
0, & u \ge 1
\end{cases}
\]

그리고 robust STL에서는 잔차가 큰 점의 영향력을 줄입니다. 이 수식들을 실무 언어로 바꾸면 다음과 같습니다.

| 수식 | 실무 해석 |
|---|---|
| \(Y_t = T_t + S_t + R_t\) | 센서 원본은 장기 변화, 반복 패턴, 남은 이상/잡음으로 구성됨 |
| \(R_t = Y_t - T_t - S_t\) | 정상적 흐름을 제거했는데도 남는 값이 이상 후보 |
| LOESS weight | 가까운 시간대 데이터를 더 믿고 부드럽게 추정 |
| robust weight | 튄 값이 trend/seasonal 추정을 망치지 않게 영향 감소 |

## 실무에서 어떻게 써야 하나?

### 1. 설비 센서 EDA의 첫 단계로 쓴다

온도, 압력, 유량, 전류 RMS, 진동 RMS 같은 센서 데이터를 받으면 먼저 원본 plot만 보지 말고 STL decomposition plot을 그려보는 것이 좋습니다.

| 확인할 것 | 질문 |
|---|---|
| trend | 장기적으로 증가/감소하는가? |
| seasonal | 일정한 운전 주기가 있는가? |
| remainder | 특정 시점에 큰 spike가 있는가? |
| component 변화 | 정비 전후로 패턴이 바뀌었는가? |

특히 설비별로 따로 그려야 합니다. 설비 A와 설비 B를 합쳐서 decomposition하면 각 설비의 고유 운전 패턴이 섞일 수 있습니다.

---

### 2. 정상 주기를 제거한 뒤 이상탐지를 한다

실무에서 흔한 실수는 “원본값이 높으면 이상”이라고 판단하는 것입니다.

하지만 원본값이 높은 이유가 정상적인 주기 때문일 수 있습니다. 따라서 다음 순서가 더 안전합니다.

1. 정상 운전 구간을 선택한다.
2. STL로 trend와 seasonal을 추정한다.
3. remainder의 정상 범위를 계산한다.
4. 새 데이터의 remainder가 정상 범위를 벗어나면 경보 후보로 본다.

예를 들어:

| 기준 | 방식 |
|---|---|
| 단순 threshold | 온도 > 80이면 경보 |
| STL 기반 threshold | trend + seasonal을 제거한 residual이 평소보다 크면 경보 |

두 번째 방식이 더 정교합니다. 특히 주야간 변동, 생산량 변동, 반복 운전 cycle이 큰 설비에서는 STL 기반 residual이 더 해석하기 쉽습니다.

---

### 3. Feature engineering에 활용한다

STL 결과를 ML 모델의 feature로 쓸 수 있습니다.

예를 들어 설비 고장 분류 모델을 만든다면 다음 feature를 만들 수 있습니다.

| feature | 의미 |
|---|---|
| trend slope | 최근 trend가 얼마나 빨리 증가하는가 |
| trend level | 현재 장기 수준이 높은가 |
| seasonal amplitude | 정상 주기 변동 폭이 커졌는가 |
| residual mean | 잔차가 한쪽으로 치우쳤는가 |
| residual std | 불규칙성이 커졌는가 |
| residual max | 순간 spike가 있었는가 |
| residual kurtosis | 충격성 이상이 있는가 |
| residual energy | 설명되지 않는 변동 에너지 |

예를 들어 베어링 진동 RMS에 STL을 적용하면, trend slope가 점점 증가하고 residual spike가 많아지는 것이 고장 전조일 수 있습니다. 다만 raw vibration waveform 자체에는 STL만 적용하기보다 FFT, envelope analysis, wavelet, spectrogram 같은 주파수 분석을 함께 사용하는 것이 보통 더 적합합니다.

---

### 4. Train/test leakage를 매우 조심한다

이 부분은 계속 반복해서 강조하겠습니다.

**STL을 train/test split 전에 전체 데이터에 적용하면 data leakage가 생길 수 있습니다.**

왜냐하면 decomposition은 주변 시점의 값을 사용해 trend와 seasonal을 추정하기 때문입니다. 특히 전체 시계열을 한 번에 STL로 분해한 뒤 그 결과 feature를 만들어 train/test를 나누면, test 구간의 정보가 train feature 생성 과정에 섞일 수 있습니다.

위험한 방식은 다음입니다.

| 단계 | 위험한 방식 |
|---|---|
| 1 | 전체 설비 데이터를 STL로 분해 |
| 2 | trend, seasonal, residual feature 생성 |
| 3 | window 단위로 랜덤 train/test split |
| 4 | accuracy 99% 보고 성능 좋다고 판단 |

이 방식은 실무 성능을 과대평가할 가능성이 큽니다.

더 안전한 방식은 다음입니다.

| 상황 | 권장 방식 |
|---|---|
| 예측 문제 | train 기간에서 STL 파라미터/패턴을 학습하고 미래 test에는 과거 정보만 사용 |
| 설비 분류 | 설비 unit 또는 run 단위로 split한 뒤, train 안에서만 전처리 기준 학습 |
| 이상탐지 | 정상 train 구간으로 baseline을 만들고, test 구간은 별도로 평가 |
| RUL | engine/bearing unit 단위 split 유지 |
| cross-validation | time-series split 또는 group split 사용 |

특히 sliding window를 만든 뒤 랜덤 split하는 것은 매우 위험합니다. 같은 설비, 같은 run, 인접 window가 train과 test에 섞이면 모델이 고장 패턴을 일반화한 것이 아니라 거의 같은 신호 조각을 외운 것일 수 있습니다.

---

### 5. 운영 모드별로 따로 분해하는 것이 좋다

설비는 운전 조건에 따라 패턴이 크게 달라집니다.

| 운전 조건 | 센서 패턴 변화 |
|---|---|
| 저부하 | 전류, 온도, 진동 낮음 |
| 고부하 | 전류, 온도, 진동 높음 |
| 예열 구간 | trend 급상승 |
| 정상 운전 | 안정적 반복 패턴 |
| 정지 직전 | 압력/유량 변화 |
| 정비 직후 | baseline 자체 변경 |

따라서 모든 운전 모드를 한꺼번에 STL로 분해하면 trend, seasonal, remainder가 어색하게 나올 수 있습니다. 가능하면 운전 모드별로 나누거나, 최소한 운전 모드 변수를 함께 기록해야 합니다.

---

## STL을 설비 진단에 적용할 때의 예시 워크플로우

### 상황

모터 전류 RMS를 1분 단위로 수집합니다. 설비는 하루 주기와 교대조 주기가 있고, 고장 전에는 전류 RMS의 baseline이 서서히 상승합니다.

### 목표

고장이 나기 전에 이상 징후를 감지합니다.

### 절차

| 단계 | 내용 |
|---|---|
| 1 | 설비별, run별로 데이터를 분리한다 |
| 2 | 결측, 센서 오류, 정비 이력, 운전 모드를 확인한다 |
| 3 | 정상 운전 구간으로 STL을 적용한다 |
| 4 | trend slope와 residual 분포를 확인한다 |
| 5 | residual이 큰 구간을 anomaly candidate로 표시한다 |
| 6 | 실제 정비 이력, 고장 로그와 비교한다 |
| 7 | false alarm과 missed detection을 계산한다 |
| 8 | 운영 threshold를 조정한다 |

### 평가 지표

이상탐지와 설비 진단에서는 accuracy 하나로 부족합니다.

| 지표 | 의미 |
|---|---|
| precision | 경보를 냈을 때 실제 이상일 확률 |
| recall | 실제 이상 중 얼마나 잡았는가 |
| F1 | precision과 recall의 균형 |
| false alarm | 정상인데 경보를 낸 경우 |
| missed detection | 이상인데 놓친 경우 |
| lead time | 고장 전에 얼마나 빨리 감지했는가 |
| confusion matrix | 정상/고장 판단의 전체 구조 |
| RUL error | 남은 수명 예측 오차 |

STL 기반 이상탐지는 특히 false alarm과 missed detection의 균형이 중요합니다. threshold를 낮추면 이상을 빨리 잡지만 false alarm이 늘고, threshold를 높이면 false alarm은 줄지만 고장을 놓칠 수 있습니다.

## 한계

### 1. 주기를 알아야 한다

STL은 seasonal period를 지정해야 합니다. 예를 들어 일별 데이터에서 주간 패턴을 보려면 period=7, 시간별 데이터에서 하루 패턴을 보려면 period=24처럼 정해야 합니다.

설비 데이터에서는 이게 쉽지 않을 수 있습니다.

| 데이터 | period 후보 |
|---|---|
| 1분 단위 온도 | 1440분 = 하루 |
| 1초 단위 압력 | 제어 cycle 길이 |
| 진동 RMS | 회전 주기 또는 운전 cycle |
| 전류 RMS | 생산 takt time 또는 교대조 주기 |

주기를 잘못 잡으면 decomposition 결과도 어색해집니다.

---

### 2. 기본 STL은 단일 seasonality 중심이다

현장 데이터는 여러 주기를 동시에 가질 수 있습니다.

예를 들어:

| 주기 | 예시 |
|---|---|
| 1분 | 제어 주기 |
| 1시간 | 생산 cycle |
| 8시간 | 교대조 |
| 24시간 | 주야간 |
| 7일 | 평일/주말 |

기본 STL은 하나의 주요 seasonal period를 다루는 데 초점이 있습니다. 여러 계절성이 강하면 MSTL 같은 확장 방법을 고려할 수 있습니다.

---

### 3. 다변량 센서 관계를 직접 학습하지 않는다

STL은 기본적으로 한 센서씩 보는 방법입니다.

하지만 설비 이상은 여러 센서의 조합에서 나타날 수 있습니다.

예를 들어:

| 센서 조합 | 이상 해석 |
|---|---|
| 전류 증가 + 유량 감소 | 펌프 막힘 가능성 |
| 온도 증가 + 진동 증가 | 마찰/베어링 문제 가능성 |
| 압력 증가 + 밸브 개도 변화 없음 | 배관 막힘 가능성 |
| 진동 특정 주파수 증가 + 전류 sideband | 회전체 결함 가능성 |

STL은 각 센서의 trend/seasonal/residual을 나누는 데는 좋지만, 센서 간 상호작용을 직접 모델링하지는 않습니다. 이런 경우에는 VAR, state-space model, autoencoder, graph neural network, Transformer 기반 multivariate anomaly detection 등을 같이 봐야 합니다.

---

### 4. 실시간 적용에서는 미래 정보 사용을 조심해야 한다

일반적인 smoothing/decomposition은 특정 시점의 값을 추정할 때 그 주변의 앞뒤 데이터를 모두 사용할 수 있습니다. 하지만 실시간 설비 모니터링에서는 미래 데이터를 아직 알 수 없습니다.

따라서 실시간 적용에서는 다음을 구분해야 합니다.

| 목적 | 허용되는 방식 |
|---|---|
| 사후 분석 | 전체 구간 STL 가능 |
| 실시간 경보 | 과거 데이터만 사용하는 causal 방식 필요 |
| 모델 평가 | test 미래 구간 정보가 train feature에 섞이면 안 됨 |
| 운영 배포 | rolling/online 방식 또는 train 기준 decomposition 필요 |

사후 리포트에서는 전체 데이터를 분해해도 괜찮지만, 예측 모델 학습이나 성능 평가에서는 매우 조심해야 합니다.

---

### 5. Raw vibration waveform에는 STL만으로 부족하다

베어링 결함, 기어 결함, 모터 편심 같은 문제는 고주파 진동 신호의 특정 주파수 성분에 나타나는 경우가 많습니다.

이런 데이터에는 STL보다 다음 기법들이 더 직접적으로 중요할 수 있습니다.

| 기법 | 용도 |
|---|---|
| FFT | 결함 주파수 성분 확인 |
| envelope analysis | 베어링 충격성 결함 검출 |
| wavelet transform | 시간-주파수 변화 확인 |
| spectrogram/STFT | 시간에 따라 변하는 주파수 패턴 확인 |
| order analysis | 회전수 변화가 있는 회전체 분석 |

다만 raw vibration을 바로 STL로 분해하기보다는, 진동 RMS, kurtosis, crest factor, 특정 주파수 band energy 같은 feature를 만든 뒤 그 feature의 trend와 residual을 STL로 보는 방식은 실무적으로 유용할 수 있습니다.

## 1편 ARIMA와 연결해서 이해하기

| 관점 | 1편 ARIMA | 2편 STL | 연결해서 이해하기 |
|---|---|---|---|
| 목적 | 미래값 예측 | 원본을 trend/seasonal/remainder로 분해 | STL로 분해한 뒤 ARIMA로 component를 예측할 수 있음 |
| 핵심 질문 | 과거 값과 과거 오차로 현재를 설명할 수 있나? | 현재 값은 추세, 반복 패턴, 잔차 중 무엇 때문인가? | ARIMA는 예측 중심, STL은 해석 중심 |
| 정상성 처리 | 차분으로 추세 제거 | trend를 직접 추정해 분리 | 차분보다 STL이 더 직관적인 경우가 있음 |
| 계절성 | SARIMA로 모델링 가능 | seasonal component로 직접 분리 | 주기가 뚜렷하면 STL이 시각적으로 이해하기 쉬움 |
| 이상탐지 | 예측 잔차 사용 | decomposition remainder 사용 | 둘 다 residual 기반 이상탐지로 연결 |
| 설비 데이터 주의점 | window random split 위험 | 전체 데이터 분해 후 split 위험 | 전처리도 train/test leakage를 만들 수 있음 |
| 실무 역할 | baseline forecasting | EDA, residual anomaly, feature engineering | 둘 다 딥러닝 전 반드시 확인할 기본 도구 |

가장 중요한 연결은 이것입니다.

**ARIMA는 “정상 패턴을 예측하고 예측 오차를 본다”는 사고이고, STL은 “정상적인 추세와 반복 패턴을 제거하고 남은 것을 본다”는 사고입니다.**

둘 다 결국 설비 이상탐지에서 residual을 중요하게 봅니다.

---

## 오늘 공부용 요약

1. STL은 시계열을 trend, seasonal, remainder로 나누는 방법이다.
2. trend는 장기적인 변화, seasonal은 반복 패턴, remainder는 설명되지 않는 잔차다.
3. 설비 센서에서는 주야간 변화, 교대조, 제어 cycle, 생산 cycle이 seasonal이 될 수 있다.
4. 원본값이 높다고 무조건 이상은 아니다. 정상적인 seasonal 때문일 수 있다.
5. STL을 쓰면 정상 반복 패턴을 제거한 뒤 residual 기반 이상탐지가 가능하다.
6. LOESS는 가까운 점을 더 많이 반영해 부드러운 선을 추정하는 방법이다.
7. STL은 inner loop에서 trend와 seasonal을 번갈아 업데이트한다.
8. outer loop에서는 이상치의 영향을 줄이는 robust weight를 사용한다.
9. train/test split 전에 전체 데이터에 STL을 적용하면 leakage가 생길 수 있다.
10. 설비 진단에서는 STL 결과를 residual feature, trend slope feature, anomaly score로 활용할 수 있다.
11. 기본 STL은 단일 seasonality 중심이므로 여러 주기가 강한 현장 데이터에서는 확장 방법이 필요할 수 있다.
12. raw vibration waveform에는 STL만으로 부족하고 FFT, envelope, wavelet 분석과 함께 봐야 한다.
